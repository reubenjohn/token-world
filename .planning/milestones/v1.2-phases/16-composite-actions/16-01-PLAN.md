Create `docs/design/composite-actions.md`. The document must contain these sections in order:

**## Overview** — one paragraph: Phase 16 adds support for one agent action firing multiple
primary mechanics within a tick (REQ-V12-ENGINE-04). Reference PROJECT.md Key Decisions as
the authoritative decision log. State that this implements v1.2 D-01.

**## Problem** — describe the limitation: today's classifier emits a single ClassifiedAction;
an action like "I open the chest and take the key" collapses to one verb, silently dropping
the second sub-action.

**## Design Options Considered (MORNING-HANDOFF §E1)** — three clearly-labelled subsections:

### Option 1: Classifier emits `actions: [...]` list (CHOSEN)
Classifier already understands sequential English ("and", "then"). Change VerdictOk to hold
`actions: list[ClassifiedAction]` instead of `classified: ClassifiedAction`. Single-verb
inputs wrap as a 1-element list for full back-compat. No new component, no extra LLM call.
Blast radius: models.py (schema change) + classifier.py (prompt + SCHEMA_VERSION) +
engine.py (iteration loop).

### Option 2: Top-K mechanic matching (REJECTED)
Rejected because `check()` may pass for accidentally-overlapping mechanics not intended
to fire together. Correctness risk without guardrails.

### Option 3: ActionDecomposer stage (REJECTED)
Rejected because it adds an extra LLM call per multi-verb action, increasing latency
and cost — violating the hobby-project budget constraint in PROJECT.md Constraints.

**## Implementation Contract** — describe the data flow:
- Classifier system prompt: emits `{"kind":"ok","actions":[{...ClassifiedAction...},...], "confidence":0.0-1.0}`
- `VerdictOk.actions: list[ClassifiedAction]` — non-empty, validated by Pydantic
- Back-compat: `@property def classified(self) -> ClassifiedAction: return self.actions[0]`
- Engine wave (Wave 2): for each sub-action in verdict.actions, run match → decide → execute;
  first YieldDecision wins the whole tick; RefuseDecision on a sub-action is recorded but
  other sub-actions still run; combined ExecutionTrace returned.
- TickSummary: adds `classified_actions: list[dict]` field (all sub-actions);
  keeps `classified_action` field (= `actions[0]`) for back-compat.

**## Key Decisions** — table with columns Decision | Value | Rationale:
| Single-action back-compat | `verdict.classified` property | Minimises diff to engine.py callers |
| Refuse-continues policy | Other sub-actions run after a refuse | Consistent with independent-action semantics |
| First-yield-wins | YieldDecision on any sub-action halts the tick | Avoids partial mechanic authoring loops |
| SCHEMA_VERSION | bumped to "2.0" | Signals prompt contract change to hash registry |

**## Non-Goals (this phase)** — list: per-sub-action diagnostics dashboard panels (v2.0),
multi-agent conflict detection (v2.0 D-18), Option 2 top-K matching, Option 3 ActionDecomposer.

```python
def _apply_known_target_check(
    self, verdict: ClassifierVerdict, known_node_ids: list[str]
) -> ClassifierVerdict:
    if not isinstance(verdict, VerdictOk):
        return verdict
    # Check each sub-action for unknown targets
    for classified in verdict.actions:
        if classified.target is not None and classified.target not in known_node_ids:
            return VerdictNoSuchTarget(target_text=classified.target)
        if (
            classified.indirect_object is not None
            and classified.indirect_object not in known_node_ids
        ):
            return VerdictNoSuchTarget(target_text=classified.indirect_object)
    return verdict
```
