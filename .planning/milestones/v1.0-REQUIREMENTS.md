## Out of Scope (original v1 decision, reaffirmed at milestone close)

| Feature | Reason |
|---------|--------|
| Game adaptation / playable game | v1 is a simulation, not a game |
| Web UI / dashboard | Prove core loop first; CLI is sufficient |
| Visual output (images, maps) | Text-first simulation |
| Real-time simulation | Turn-based is simpler and sufficient |
| Civic simulation scenarios | Requires mature mechanics ecosystem |
| Contributor onboarding / CI/CD | Premature until core is stable (reversed during v1.0 — basic CI via ruff + mypy + pytest did land) |
| Distributed graph / sharding | Graph will be small for years |
| Authentication / multi-user | Local hobby project |
| Plugin system for mechanics | Framework IS the plugin API |

## Changed Requirements During v1.0

- **MECH-03 "LLM generates valid Python mechanics":** Reframed in Phase 4 from a bespoke generation pipeline to operator-driven SDLC (D-34, inversion of control). Rationale: the top-level coding agent (Opus via Agent SDK) already IS a code-authoring agent with better tooling than anything bespoke would offer; the framework's job shifted from "generate" to "validate + yield-to-operator when needed."
- **SIM-03 "Engine triggers mechanic generation when no match":** Reframed to "Engine yields to operator with structured YieldSignal"; resume_tick picks up new mechanic once operator completes SDLC. Same end-user outcome, different architectural locus.
- **MECH-05/MECH-06 (mechanic layout + registry):** Originally Phase 2 folder-per-mechanic (D-15); revised in Phase 4 to flat-Python layout (D-03..D-08). Phase 2 seeds were migrated in Plan 04-01.
- **UNIV-03 (MCP surface):** Originally broader in Phase 0; narrowed in Phase 4 to 3-tool surface (resume_tick, rollback, list_mechanics) per D-19. Operator uses filesystem + SQLite directly for inspection rather than MCP RPC.
- **Phase 04.1 added (Operator Agent Harness):** Urgent inserted phase; not in original roadmap. Required by Phase 5 yield-contract verification.
- **Phase 07.1 added (claude-cli LLM backend):** Urgent inserted phase; added to unblock Phase 6 live-API UAT at $0 marginal cost. Created the post-hoc INFRA-01 requirement.
