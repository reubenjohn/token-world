<objective>
Wave-0 single-task prep. Expose the projection dict that the Observer synthesized against as a
first-class field on `TickResult`, so Phase 6 Plan 05 (TurnScorer) can compute
`observation_groundedness` by checking whether the observation text references node IDs from
the projection — WITHOUT reaching into `engine._projector` (private attribute access) and without
re-projecting after the tick (duplicate work, potential drift).

This is the one recommendation in 06-RESEARCH.md §"Open Questions" that calls for a small
engine.py + models.py change. Keeping it in its own Wave-0 plan isolates the engine modification
from the three parallel Wave-1 streams (resident, compressor, regression) so they can all run
in parallel without touching engine/engine.py.
</objective>

<context>
Prior deliverables:
- Phase 5 `SimulationEngine.run_tick(action_text, actor) -> TickResult` (05-08)
- Phase 5 `TickResult` dataclass (engine/engine.py) with `ok/yielded/refused` classmethods
- Phase 5 `VisibilityProjector.project_for(actor) -> dict[str, dict]` (05-04)

Downstream consumers (this plan unblocks):
- 06-05 TurnScorer uses `result.projected_state` keys as the groundedness substring-check set
  per 06-RESEARCH §"Groundedness check detail" and D-12 metric #2

Execute path in engine.py _handle_execute currently calls:
    projection = self._projector.project_for(actor)
    observation = self._observer.synthesize(projection=projection, ...)
We want that same `projection` attached to the returned TickResult without re-computing.

Contracts (what downstream plans will consume from this plan):
```python
@dataclass(frozen=True, slots=True)
class TickResult:
    kind: str
    tick_id: str
    observation: str | None = None
    yield_signal: YieldSignal | None = None
    trace: ExecutionTrace | None = None
    refusal_reason: str | None = None
    projected_state: dict | None = None  # NEW

    @classmethod
    def ok(cls, *, tick_id, observation, trace, projected_state=None) -> TickResult: ...
```
