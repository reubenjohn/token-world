<objective>
Close the quality-infrastructure loop for Phase 6 by delivering:

1. `token-world playtest <slug>` CLI command that drives N-turn runs, writes structured
   JSON reports, supports YAML scenarios with adversarial injection, and handles engine
   yields via the operator harness.
2. `TurnScorer` implementing the five deterministic rubric metrics from D-12 (mechanic
   match, observation groundedness, mutation count, refusal rate, action novelty).
3. `Scenario` YAML loader with `InjectionSampler` covering D-11 injection types.
4. `PlaytestReport` Pydantic model matching D-23 schema.

Bundled into one plan because runner + scorer are tightly coupled (runner feeds results to
scorer; they share result types) and splitting them across parallel plans would create a
file-overlap on `playtest/runner.py`.

Hook points are left open for Waves 3:
- `hash_check_fn` → plugged by 06-05 (prompt hash registry + regression trigger)
- `injection_sampler_factory` → extended by 06-06 (adversarial bank YAML + sampler config)

- `src/token_world/playtest/` package (5 modules: __init__, runner, scorer, scenarios, report)
- `scenarios/example.yaml` canonical example
- `token-world playtest` Click command

</objective>

PlaytestReport schema (D-23, Pydantic):

```python
class TurnRecord(BaseModel):
    turn_number: int
    action_text: str
    observation_text: str | None
    tick_id: str
    kind: str   # "ok" | "yielded" | "refused"
    score: dict  # TurnScore.model_dump()

class AggregateScores(BaseModel):
    mechanic_match_rate: float
    observation_groundedness: float
    mutation_count: float
    refusal_rate: float
    action_novelty: float
    composite: float

class PlaytestReport(BaseModel):
    run_id: str              # uuid or timestamp
    scenario_file: str | None
    turns: list[TurnRecord]
    aggregate_scores: AggregateScores
    prompts_sha256: dict[str, str]   # populated by 06-05 hook; empty dict by default
    duration_ms: int
    schema_version: Literal[1] = 1
```

Discretion (D-28, D-30 locked in this plan):
- **D-28 progress format**: plain stdout, one line per turn, no rich/tqdm. CI-friendly.
- **D-30 novelty metric**: cosine on collections.Counter word-bag over last 3 action texts.
  Implementation: normalize lowercased word counts, compute dot product / (norm_a * norm_b).
  Returns 1.0 for first turn (no history). No numpy/scipy — stdlib only.
