# Phase 06: Code Review Report

**Reviewed:** 2026-04-13
**Depth:** standard
**Files Reviewed:** 22
**Status:** issues_found

## Summary

Phase 06 delivers three interlocking capabilities: the ResidentAgent (personality, memory, session forking), TickCompressor (online batch/epoch compression), and the playtest quality infrastructure (runner, scorer, adversarial bank, hash registry, Sonnet judge). The code is well-structured, correctly uses parameterized SQLite queries throughout, respects the KnowledgeGraph API boundary, and implements atomic write-then-delete crash safety in the compressor. No SQL injection or hardcoded credentials were found.

Four issues warrant attention before the phase is considered production-ready:

1. The `--output` path override in the PlaytestRunner is silently dropped — the caller's named path is never used.
2. `PlaytestRunner` never calls `memory.maybe_compact_summary()`, so the rolling memory summary is never updated during multi-turn playtest runs.
3. `ResidentAgent.run_turn()` does not guard against an empty `response.content` list, meaning an empty LLM response causes an unhandled `IndexError`.
4. `PromptHashRegistry.trigger_regression` runs `uv run pytest` as a subprocess with no working-directory pin — cwd at call time determines which project's tests run.

## Warnings

### WR-01: `--output` path is silently ignored by PlaytestRunner

**File:** `src/token_world/playtest/runner.py:185-188`
**Issue:** When `output_path` is provided the runner calls `report.write(output_path.parent)`, which generates a new UUID-named file in `<parent>/playtest-reports/<run_id>.json`. The caller-specified filename is completely discarded. A user passing `--output /tmp/my_run.json` gets a file at `/tmp/playtest-reports/<uuid>.json` instead — the intended destination is never written.

**Fix:**
```python
# In runner.py, replace the output block:
if output_path is None:
    return report.write(universe_dir)
else:
    # Honour the exact path the caller specified
    output_path.parent.mkdir(parents=True, exist_ok=True)
    from token_world.mechanic.diagnostics import _atomic_write_json
    _atomic_write_json(output_path, report.model_dump())
    return output_path
```

### WR-02: `PlaytestRunner` never calls `memory.maybe_compact_summary()`

**File:** `src/token_world/playtest/runner.py:114-167`
**Issue:** The rolling memory summary (D-07) is regenerated by `AgentMemory.maybe_compact_summary()` at every 10-turn boundary, but `PlaytestRunner.run()` never calls it. The `agent-turn` CLI command (cli.py:1175) calls it correctly. In a 20-turn playtest the summary will never be written, so turns 11+ operate with a stale or absent memory summary. This means the agent's context degrades silently during the primary demo scenario.

**Fix:** Call `maybe_compact_summary` after each `memory.store_turn` in the runner loop:
```python
# After memory.store_turn(...) in runner.py, around line 142:
self.memory.store_turn(
    self.agent_id, self.session_id, turn_num,
    action, result.observation or "", result.tick_id,
)
# Add:
self.memory.maybe_compact_summary(  # type: ignore[attr-defined]
    self.session_id, self.engine._anthropic_client  # type: ignore[attr-defined]
)
```
Alternatively, expose the `client` as a `PlaytestRunner` field so the call does not require access to a private engine attribute.

### WR-03: `run_turn()` crashes with `IndexError` on empty LLM response

**File:** `src/token_world/resident/agent.py:103`
**Issue:** `return str(response.content[0].text).strip()` assumes `response.content` is non-empty. If the Anthropic API returns an empty content list (malformed response, streaming truncation, or stop-sequence hit with no output), this raises `IndexError` with no user-visible error message. The same unguarded pattern exists in `memory.py:203` and `personality.py:87`.

**Fix:**
```python
# In agent.py run_turn():
content = response.content
if not content:
    raise ValueError("LLM returned empty response in run_turn()")
return str(content[0].text).strip()
```
Apply the same guard in `memory.py:203` and `personality.py:87`.

### WR-04: `trigger_regression` subprocess has no working-directory pin

**File:** `src/token_world/playtest/hash_registry.py:175-202`
**Issue:** `subprocess.run(["uv", "run", "pytest", "tests/test_regression/", ...])` runs with whatever `cwd` the calling process has at invocation time. If the runner is invoked from a directory other than the project root (e.g., from inside a universe directory), the pytest command will fail to find `tests/test_regression/` or — if another `pyproject.toml` is in scope — may run against an unrelated project. There is no `cwd=` argument.

**Fix:**
```python
# In hash_registry.py trigger_regression, add cwd discovery:
import importlib.resources
import token_world
project_root = Path(token_world.__file__).parent.parent.parent

result = subprocess.run(
    cmd,
    capture_output=True,
    text=True,
    timeout=600,
    check=False,
    cwd=project_root,  # pin to project root
)
```
A simpler alternative: resolve via `Path(__file__)` — `hash_registry.py` lives at `src/token_world/playtest/`, so `Path(__file__).parents[3]` gives the project root.

## Info

### IN-01: `memory.new_summary` uses `response.content[0].text` without cast

**File:** `src/token_world/resident/memory.py:203`
**Issue:** `new_summary = response.content[0].text.strip()` returns `Any` (Anthropic SDK typing), while the target column `memory_summary TEXT` expects a `str`. The `agent.py` deviation log notes this was fixed for `run_turn()` via `str(...)`, but the identical pattern in `memory.py` was left uncast. Mypy may flag this.
**Fix:** `new_summary = str(response.content[0].text).strip()`

### IN-02: `_build_messages` duplicates "What do you do next?" when `turns` exist and `summary` is non-empty

**File:** `src/token_world/resident/agent.py:151-161`
**Issue:** The conditional logic is:
```python
if not turns and summary:
    ...append summary+prompt...
else:
    ...append plain "What do you do next?"
```
When `turns` is non-empty and `summary` is non-empty (the common long-session case), the `else` branch fires and the summary is silently dropped. The summary was already prepended to the first turn's user message at line 147, so this is not a data loss bug — but the conditional structure is confusing and the comment on line 152 states "No history yet but there's a summary" without explaining that the summary is already embedded in the turns block when turns exist.
**Fix:** Add a clarifying comment or restructure to make the "summary in turns vs. summary standalone" logic explicit.

### IN-03: `adverarial_rate` scenario field is parsed but never applied

**File:** `src/token_world/playtest/scenarios.py:77`, `src/token_world/playtest/runner.py`
**Issue:** `Scenario.adversarial_rate` is loaded from YAML and stored, but neither `PlaytestRunner` nor `_determine_action` reads it. D-11 specifies "injection rates are configurable per scenario (`adversarial_rate: 0.1`)" as a feature of free-turn auto-injection. Currently the field is inert. This is not a correctness bug (the runner still works) but the advertised feature is silently unimplemented.
**Fix:** In `_determine_action`, when `kind == "agent"` and `scenario.adversarial_rate > 0`, roll a uniform sample against `adversarial_rate` and substitute an adversarial injection when it fires.
