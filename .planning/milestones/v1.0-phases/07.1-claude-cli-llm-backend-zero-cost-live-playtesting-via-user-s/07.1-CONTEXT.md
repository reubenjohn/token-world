## Phase Boundary

Phase 07.1 delivers a **pluggable LLM backend** that routes simulation LLM calls through either:
- **AnthropicSDKBackend** (default, existing behavior â€” raw `anthropic.Anthropic` SDK)
- **ClaudeCLIBackend** (new â€” subprocess wrapper around `claude --model <id> -p <prompt>`)

**Motivation:** Phase 6 VERIFICATION has 3 `human_needed` UAT items that require live LLM calls. The raw SDK path costs ~$0.50 per 5-turn playtest Ă— 3 scenarios = ~$1.50+. The user's Claude subscription already includes Haiku/Sonnet usage. A subprocess wrapper around the `claude` CLI lets the simulation hit those models **at zero marginal cost**, unlocking live UAT for v1.0 milestone close.

**What this phase delivers:**
1. `src/token_world/engine/llm_backend.py` â€” new module with `LLMBackend` Protocol, `AnthropicSDKBackend`, `ClaudeCLIBackend`, `get_backend()` env-dispatched factory, and `_strip_markdown_fences()` helper.
2. Refactor `Classifier` (engine/classifier.py), `Observer` (engine/observer.py), `ResidentAgent` (resident/agent.py) to accept an optional `backend: LLMBackend | None = None` parameter alongside the existing `client` field (backward-compatible).
3. Unit tests for backend (mocking `subprocess.run` for CLI path).
4. Integration smoke test: `TOKEN_WORLD_BACKEND=claude-cli` routes through a mocked subprocess that returns canned JSON; the existing classifier/observer logic handles it identically to SDK-path responses.
5. Docs: `docs/guides/claude-cli-backend.md` explaining env var, model ID requirements (`claude-haiku-4-5`, `claude-sonnet-4-5` â€” full IDs required; aliases like `haiku-4-5` DO NOT work), latency expectations (~10-15s/call), and how to run UAT.

**Explicitly OUT of scope:**
- Token/cost telemetry via CLI (CLI does not expose `input_tokens`/`output_tokens`; reports show `$0.00 (via CLI subscription)` for CLI-backend runs).
- Streaming â€” both backends are synchronous block-return.
- Rate-limit handling â€” the CLI path inherits the user's subscription limits; failures surface as CalledProcessError.
- Session/memory passthrough â€” each `-p` invocation is fresh; no `--continue` / session reuse in v1.
- Multi-backend per-call (e.g., classifier via CLI, observer via SDK) â€” `TOKEN_WORLD_BACKEND` is global.

## Implementation Decisions

### D-01 â€” One Protocol, Two Implementations, Env-Var Dispatch
*Source: MORNING-HANDOFF Â§5; prior-phase pattern (Protocols in `engine/long_running.py`, `mechanic/matchers.py`).*
A single `LLMBackend` Protocol with one method â€” `call(*, model: str, system: str, prompt: str, max_tokens: int) -> str`. Two concrete implementations: `AnthropicSDKBackend` (wraps existing `anthropic.Anthropic` client) and `ClaudeCLIBackend` (subprocess wrapper). Factory `get_backend()` reads `TOKEN_WORLD_BACKEND` env var: `"claude-cli"` â†’ CLI; anything else (or unset) â†’ SDK. Rejected: dual constructors, per-call backend selection, config-file dispatch. Rationale: env var is the standard pattern for "same code, different runtime" (matches `ANTHROPIC_API_KEY` conventions).

### D-02 â€” Backward-Compatible Constructor Refactor
*Source: existing `Classifier` / `Observer` / `ResidentAgent` signatures; preserve 1640+ test baseline.*
Add `backend: LLMBackend | None = None` parameter to each LLM-calling class. Existing `client: Any` field stays. `__post_init__` / `__init__` logic: if `backend is None`, wrap `self.client` in `AnthropicSDKBackend(self.client)` if `self.client is not None`, else call `get_backend()`. Internal `_send()` methods (or equivalents) route through `self.backend.call(...)` instead of `self.client.messages.create(...)`. Existing tests inject FakeClient â†’ auto-wrapped â†’ still work. New tests can inject a `FakeBackend` directly OR set `TOKEN_WORLD_BACKEND=claude-cli` + mock subprocess. Rejected: replacing `client` field entirely (breaks all 1640 tests at once). Rationale: additive change preserves test baseline; `client` path is the "fake Anthropic SDK" escape hatch that tests already rely on.

### D-03 â€” Markdown Fence Stripping is a Backend Responsibility
*Source: live CLI validation 2026-04-14 â€” CLI wraps JSON output in ```json ... ``` fences even when explicitly asked not to.*
`ClaudeCLIBackend.call()` strips leading/trailing ``` fences before returning text. Isolated helper `_strip_markdown_fences(text)` handles both ```json\n...\n``` and plain ```\n...\n```. If input doesn't start with ```, returned verbatim. SDK backend does NOT strip (Anthropic SDK returns unfenced text for this use case). Rationale: keeps fence-handling at the adapter boundary â€” classifier/observer downstream see unified clean text.

### D-04 â€” Full Model IDs Required, No Alias Translation
*Source: live CLI validation 2026-04-14 â€” `claude --model haiku-4-5` returns "model may not exist"; `claude --model claude-haiku-4-5` works.*
All model strings passed to `ClaudeCLIBackend.call(model=...)` MUST be full IDs (`claude-haiku-4-5-20251001`, `claude-sonnet-4-5`, etc.). No alias rewriting in the backend â€” callers already pass full IDs. Tested: `claude-haiku-4-5-20251001` works; `claude-sonnet-4-5` works. Rationale: the simulation's `_MODEL` constants are already full IDs; no translation layer needed.

### D-05 â€” Subprocess Timeout 120s, check=True
*Source: live CLI validation 2026-04-14 â€” classifier-style prompt took 14.6s; observer prompts may be longer.*
`subprocess.run(..., timeout=120, check=True, capture_output=True, text=True)`. 120s handles normal CLI latency (~15s) with 8Ă— safety margin for transient slowdowns. `check=True` raises `CalledProcessError` on non-zero exit, which propagates up the stack (caller's existing `ValidationError`/retry logic handles the failure). Rejected: `check=False` + status-code inspection (more complex; adds implicit fallback paths). Rationale: loud failure beats silent degrade â€” same posture as other phase-7 assertions (D-09 safe defaults).

### D-06 â€” No Subprocess Executable Path Lookup
*Source: MORNING-HANDOFF Â§5 starter code.*
`subprocess.run` uses `["claude", "--model", model, "-p", full]` as arg list â€” relies on `claude` being on `PATH`. Documented in the guide. Rejected: `shutil.which("claude")` with explicit fallback to full path. Rationale: standard CLI tool invocation convention; if `claude` is missing, `subprocess.run` raises `FileNotFoundError` immediately â€” that's the right failure mode.

### D-07 â€” Cost Telemetry Best-Effort Word-Count Estimate
*Source: MORNING-HANDOFF Â§5 note on token telemetry.*
When running under CLI backend, `TickSummaryWriter` and `TurnScorer` cost fields report `$0.00` with annotation `"via CLI subscription"`. Token counts use `len(prompt.split()) + len(response.split())` Ă— 1.3 as a rough proxy (not exposed through `claude -p`). The existing SDK path continues to use `response.usage.input_tokens` / `output_tokens` when available. Rejected: parse `claude -p --json` (format not stable across versions), omit telemetry entirely (breaks existing report schema). Rationale: preserving existing schema fields with best-effort values is less disruptive than schema evolution.

### D-08 â€” `TOKEN_WORLD_BACKEND` Env Var is the Only Switch
*Source: MORNING-HANDOFF Â§5.*
No CLI flag on `token-world playtest`; no config-file entry. Users set `TOKEN_WORLD_BACKEND=claude-cli` in their shell once. Rejected: `--backend` flag on every CLI command (proliferates across 4 commands), `~/.config/token-world.yaml` entry (new config format). Rationale: one env var is the minimum viable surface; composes cleanly with `token-world playtest <slug>` invocations.

### D-09 â€” SubProcess Path is `subprocess.run` Not `Popen`
*Source: MORNING-HANDOFF Â§5.*
Synchronous `subprocess.run` â€” simplest, sufficient for v1. No streaming, no concurrent calls. Rejected: `asyncio.subprocess` / `Popen` + polling (adds async complexity; simulation is already synchronous). Rationale: matches the rest of the engine's synchronous design.

### D-10 â€” Backend Injected via Constructor, Not Module-Level Singleton
*Source: Classifier/Observer/ResidentAgent D-01 (Phase 5 D-01) â€” test mocking via constructor injection.*
Each class gets `backend: LLMBackend | None = None` in its constructor. If None, defaults via `get_backend()`. This preserves Phase 5 D-01's pattern: "client injected via constructor (not module-level) so test mocking works without patching." Rejected: module-level `_BACKEND = get_backend()` + import-time binding (breaks test isolation). Rationale: dependency injection is the project's established pattern.

### Claude's Discretion

- Backend unit tests use `unittest.mock.patch("subprocess.run")` for CLI path; no real `claude` invocation in unit tests. Integration smoke test (opt-in with marker `live_cli`) invokes real `claude -p`.
- Guide doc: `docs/guides/claude-cli-backend.md` â€” ~1 page, covers env var, latency, model IDs, known-format fence-stripping detail, troubleshooting.
- No REQUIREMENTS.md additions needed â€” this is infrastructure, not a new capability. (If the verifier wants a req ID, can coin `INFRA-01` post-hoc.)

## Backend Starter Code (from MORNING-HANDOFF Â§5, augmented with D-03/D-04/D-05)

```python
# src/token_world/engine/llm_backend.py
import os
import subprocess
from typing import Any, Protocol


class LLMBackend(Protocol):
    def call(self, *, model: str, system: str, prompt: str, max_tokens: int) -> str: ...


def _strip_markdown_fences(text: str) -> str:
    text = text.strip()
    if not text.startswith("```"):
        return text
    first_newline = text.find("\n")
    if first_newline == -1:
        return text
    text = text[first_newline + 1:]
    if text.endswith("```"):
        text = text[:-3]
    return text.strip()


class AnthropicSDKBackend:
    def __init__(self, client: Any) -> None:
        self._client = client

    def call(self, *, model: str, system: str, prompt: str, max_tokens: int) -> str:
        resp = self._client.messages.create(
            model=model,
            max_tokens=max_tokens,
            system=system,
            messages=[{"role": "user", "content": prompt}],
        )
        return resp.content[0].text if resp.content else ""


class ClaudeCLIBackend:
    def __init__(self, timeout: int = 120) -> None:
        self._timeout = timeout

    def call(self, *, model: str, system: str, prompt: str, max_tokens: int) -> str:
        # max_tokens is advisory only â€” CLI does not accept max_tokens argument
        full = f"{system}\n\n{prompt}"
        result = subprocess.run(
            ["claude", "--model", model, "-p", full],
            capture_output=True,
            text=True,
            timeout=self._timeout,
            check=True,
        )
        return _strip_markdown_fences(result.stdout)


def get_backend() -> LLMBackend:
    backend_env = os.environ.get("TOKEN_WORLD_BACKEND", "").strip().lower()
    if backend_env == "claude-cli":
        return ClaudeCLIBackend()
    from anthropic import Anthropic
    return AnthropicSDKBackend(Anthropic())
```

## Refactor Pattern (applied uniformly to Classifier, Observer, ResidentAgent)

Before:
```python
@dataclass(slots=True)
class Classifier:
    client: Any
    model: str = _MODEL
    max_tokens: int = 1024

    def _send(self, user_prompt, *, tick_diag_ctx, attempt=1):
        response = self.client.messages.create(
            model=self.model, max_tokens=self.max_tokens,
            system=_SYSTEM_PROMPT,
            messages=[{"role": "user", "content": user_prompt}],
        )
        raw = response.content[0].text if response.content else ""
        ...
```

After:
```python
@dataclass(slots=True)
class Classifier:
    client: Any = None  # deprecated; kept for backward compatibility
    model: str = _MODEL
    max_tokens: int = 1024
    backend: LLMBackend | None = None

    def __post_init__(self):
        if self.backend is None:
            if self.client is not None:
                object.__setattr__(self, "backend", AnthropicSDKBackend(self.client))
            else:
                object.__setattr__(self, "backend", get_backend())

    def _send(self, user_prompt, *, tick_diag_ctx, attempt=1):
        raw = self.backend.call(
            model=self.model,
            system=_SYSTEM_PROMPT,
            prompt=user_prompt,
            max_tokens=self.max_tokens,
        )
        ...
```

Note: `object.__setattr__` is needed because `dataclass(slots=True)` generates frozen-like behavior only when `frozen=True`, but the pattern is defensive against future `frozen=True` flips. Plain `self.backend = ...` also works for non-frozen slots dataclasses; executor's choice.

**For ResidentAgent** (not a dataclass; uses regular `__init__`): add `backend: LLMBackend | None = None` kwarg; store as `self._backend`; wrap `self._client` if `backend is None` and `self._client` is set.
