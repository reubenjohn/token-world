# Phase 07.1 Plan 02: Classifier + Observer + ResidentAgent LLMBackend refactor Summary

One-liner: Wired the LLMBackend abstraction (Plan 01) into the three simulation LLM-calling classes with a backward-compatible wrap-or-default pattern, preserved Observer D-24 token telemetry via an Observer-local `_UsageCapturingSDKBackend` subclass, and proved end-to-end that `TOKEN_WORLD_BACKEND=claude-cli` now routes Classifier / Observer / ResidentAgent through `subprocess.run` for zero-cost live UAT.

## What Shipped

### 1. Wrap-or-default pattern (3 instances, identical shape)

Each LLM-calling class now accepts a `backend: LLMBackend | None = None` constructor parameter and runs this logic at construction time (D-02, D-10):

```
if backend is not None:         -> use injected backend verbatim
elif client is not None:         -> wrap client in AnthropicSDKBackend
else:                            -> call get_backend() (env-var dispatch)
```

| Class           | Storage       | Wrap class                        | Location                                                |
| --------------- | ------------- | --------------------------------- | ------------------------------------------------------- |
| `Classifier`    | `self.backend` (via `__post_init__` + `object.__setattr__` for frozen-safety) | `AnthropicSDKBackend` (plain) | `src/token_world/engine/classifier.py` |
| `Observer`      | `self.backend` (via `__post_init__`) | `_UsageCapturingSDKBackend` (Observer-local subclass) | `src/token_world/engine/observer.py` |
| `ResidentAgent` | `self._backend` (direct `__init__` attrs â€” non-dataclass) | `AnthropicSDKBackend` (plain) | `src/token_world/resident/agent.py`    |

The `client` field / parameter is preserved on all three classes as the fake-SDK escape hatch existing tests rely on (D-02: "existing `client: Any` field stays"). All three `client` parameters now default to `None`.

### 2. Observer token-capture preservation â€” `_UsageCapturingSDKBackend`

The most subtle piece. Plan 01's `AnthropicSDKBackend.call()` returns only the response text; the SDK response object (with `.usage.input_tokens` / `.output_tokens`) is consumed inside the wrapper. Observer's existing D-24 test (`test_token_usage_captured_after_call`) asserts `observer.last_input_tokens == 100` â€” a non-trivial value â€” so zero-degrading SDK token capture was NOT acceptable.

Solution: a subclass of `AnthropicSDKBackend` lives inside `observer.py` (not `llm_backend.py` â€” Plan 01 owns that module):

```python
class _UsageCapturingSDKBackend(AnthropicSDKBackend):
    def __init__(self, client):
        super().__init__(client)
        self._last_usage: dict[str, int] | None = None

    def call(self, *, model, system, prompt, max_tokens) -> str:
        resp = self._client.messages.create(...)
        usage = getattr(resp, "usage", None)
        if usage is not None:
            self._last_usage = {
                "input_tokens": int(getattr(usage, "input_tokens", 0)),
                "output_tokens": int(getattr(usage, "output_tokens", 0)),
            }
        else:
            self._last_usage = None
        ...
```

`Observer.__post_init__` uses this subclass for the auto-wrap path. `synthesize()` reads `self.backend._last_usage` after each call and populates `self.last_input_tokens` / `self.last_output_tokens`. This preserves D-24 telemetry for the SDK path while silently degrading to 0 for CLI / direct-injection paths â€” exactly what CONTEXT D-07 specifies ("CLI subscription = no token visibility"). No changes to `llm_backend.py`. No expansion of the Plan 01 contract.

**Explicitly NOT done (in-scope per plan):** the CLI-path word-count token estimate from CONTEXT D-07 is NOT wired here â€” counters stay at 0 for CLI. D-07 is documented as OUT OF SCOPE for Plan 02 per the plan's falsehoods list; report-schema preservation is a follow-up.

### 3. ResidentAgent message flattening

`ResidentAgent.run_turn()` previously called `self._client.messages.create(messages=[...])` with a multi-turn user/assistant array built by `_build_messages()`. `LLMBackend.call()` takes a single `prompt: str` parameter â€” no messages array â€” so the history is now flattened:

```python
prompt = "\n\n".join(f"{m['role']}: {m['content']}" for m in messages)
text = self._backend.call(model=..., system=..., prompt=prompt, max_tokens=256)
```

For the SDK auto-wrap path this is a SEMANTIC CHANGE: the wrapped SDK still issues `messages.create(...)`, but the `messages` array now contains exactly one user-role block whose content is the flattened prior-turns transcript + final `"What do you do next?"` prompt.

The raw payload still reaches the LLM; the role structure is just encoded inline.

Empty-response handling also switched from `if not content: raise` (against the SDK response object's content list) to `if not text or not text.strip(): raise` (against the backend's returned string). WR-03 regression test (`test_run_turn_raises_on_empty_content_list`) still passes â€” the `AnthropicSDKBackend.call()` path returns `""` for empty content, which the `not text or not text.strip()` branch catches.

### 6. The `TOKEN_WORLD_BACKEND=claude-cli` end-to-end thread

Plan 01 proved the backend module works in isolation. Plan 02 proves the env var actually reaches the LLM-calling classes:

```
User shell: export TOKEN_WORLD_BACKEND=claude-cli
  â””â”€> Classifier() / Observer() / ResidentAgent() constructed with no client/backend
        â””â”€> __post_init__ / __init__ calls get_backend()
              â””â”€> get_backend() reads env var, returns ClaudeCLIBackend()
                    â””â”€> self.backend = ClaudeCLIBackend
                          â””â”€> classify() / synthesize() / run_turn() calls self.backend.call()
                                â””â”€> subprocess.run(["claude", "--model", ..., "-p", ...])
```

All three paths verified (one test per class) plus one test showing `_strip_markdown_fences` downstream parses ``` fenced JSON cleanly. Phase 07.1 is now live-UAT-ready:

```
TOKEN_WORLD_BACKEND=claude-cli token-world playtest <slug>
```
