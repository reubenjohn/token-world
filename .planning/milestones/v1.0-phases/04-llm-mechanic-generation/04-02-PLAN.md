<objective>
Deliver the validation gate that every mechanic must pass before the registry loads it. Under inversion of control (D-01/D-02), this is how the framework enforces authoring discipline — AST rules + class-contract + test-execution + dry-execute smoke. Both CLI (`validate-mechanic`) and registry auto-scan consume the single `validate(module_path)` implementation (D-12).

Purpose: MECH-04 is the phase's biggest safety net. Without it, Phase 5 cannot trust the registry. Additionally satisfies the Phase-4 portion of GAP-ENG16 per D-34 — the validation gate rejects garbage authored mechanics, obviating any "manual-review queue".

class ValidationFinding:
    stage: str        # "syntax" | "ast" | "import" | "contract" | "tests" | "smoke"

    severity: str     # "error" | "warning"

    path: str

    col: int | None = None

@dataclass
class ValidationReport:
    module_path: Path
    findings: list[ValidationFinding]
    passed: bool

def validate(module_path: Path) -> ValidationReport:
    """Run 6-stage pipeline. Stop at first hard-failing stage; accumulate
    all findings within a stage before stopping. Warnings never stop the pipeline."""

Single entry `validate(module_path)` runs the 6-stage pipeline per D-12/D-13.
AST rules from D-14. Reports structured diagnostics per D-16.

from __future__ import annotations

import ast
import importlib.util
import inspect
import json
import subprocess
import sys
from dataclasses import asdict, dataclass, field
from pathlib import Path

FORBIDDEN_CALL_NAMES: frozenset[str] = frozenset({
    "eval", "exec", "__import__", "compile", "globals", "open",
})
FORBIDDEN_IMPORT_PREFIXES: tuple[str, ...] = (
    "networkx",
    "token_world.graph.knowledge_graph",
)

Step B — Dataclasses `ValidationFinding` and `ValidationReport` per the `<interfaces>` section. `ValidationReport.fail(finding)` appends and sets `passed=False` only on `severity=="error"`. Add `ValidationReport.to_dict(self) -> dict` returning `{"module_path": str, "passed": bool, "findings": [asdict(f) for f in findings]}` for JSON output / diagnostics writes.

Step C — `_MechanicAstVisitor(ast.NodeVisitor)`. Override `visit_Import`, `visit_ImportFrom`, `visit_Call`, `visit_ClassDef` per RESEARCH.md Pattern 2. Key points:
- `visit_Import`: for each alias, forbid if `alias.name == p or alias.name.startswith(p + ".")` for any `p` in `FORBIDDEN_IMPORT_PREFIXES`.
- `visit_ImportFrom`: check `node.module` against same prefixes.
- `visit_Call`: only flag `ast.Name(id=name)` where `name in FORBIDDEN_CALL_NAMES`. Do NOT flag attribute calls like `foo.eval` — only bare-name calls per D-14.
- `visit_ClassDef`: detect any class with a base named "Mechanic" (`ast.Name(id="Mechanic")`); append to `self.mechanic_classes`. Visit children via `self.generic_visit(node)`.
- Accumulate findings in `self.errors` list; CALLER walks entire tree before checking.

Step D — Stage functions `_stage_syntax`, `_stage_ast`, `_stage_import`, `_stage_contract`, `_stage_tests`, `_stage_smoke`. Each takes `(report: ValidationReport, ...state...)` and mutates `report`. Compose in `validate()`:

```python
def validate(module_path: Path) -> ValidationReport:
    report = ValidationReport(module_path=module_path, findings=[], passed=True)
    tree = _stage_syntax(report, module_path)
    if not report.passed:
        return report
    _stage_ast(report, tree, module_path)
    if not report.passed:
        return report
    module = _stage_import(report, module_path)
    if not report.passed:
        return report
    mechanic_classes = _stage_contract(report, module)
    if not report.passed:
        return report
    _stage_tests(report, module_path, mechanic_classes)
    if not report.passed:
        return report
    _stage_smoke(report, mechanic_classes)
    return report
```

Step D1 — `_stage_syntax(report, module_path)`: read file with `module_path.read_text(encoding="utf-8")`; `ast.parse(source, filename=str(module_path))`. On `SyntaxError`, fail with line=e.lineno, col=e.offset. Return the parsed tree (or None on failure).

Step D2 — `_stage_ast(report, tree, module_path)`: run `_MechanicAstVisitor(module_path)`, visit, copy visitor's errors to report. If `len(visitor.mechanic_classes) == 0`, append a WARNING (severity="warning", rule="no_mechanic_subclass") — does NOT flip `passed`.

Step D3 — `_stage_import(report, module_path)`: use `importlib.util.spec_from_file_location(f"_validate_{module_path.stem}_{id(module_path)}", module_path)`; `importlib.util.module_from_spec`; `spec.loader.exec_module(module)`. Wrap in try/except Exception — on any raise, fail with rule="import_failed", message=`f"{type(e).__name__}: {e}"`. Return the module.

Step D4 — `_stage_contract(report, module)`: import `Mechanic` from `token_world.mechanic.protocol`. Collect classes: `[c for _, c in inspect.getmembers(module, inspect.isclass) if issubclass(c, Mechanic) and c is not Mechanic and c.__module__ == module.__name__]`. If empty → fail (rule="no_mechanic_subclass", severity="error"). For each class verify:
- `hasattr(cls, "id")` and `isinstance(cls.id, str)` and `cls.id != ""` → else fail with appropriate rule.
- `hasattr(cls, "description")` and `isinstance(cls.description, str)` → else fail.
- `hasattr(cls, "check")` and `callable(cls.check)` and `list(inspect.signature(cls.check).parameters) == ["self", "ctx"]` → else fail. Accept `inspect.signature` raising TypeError and treat as fail.
- Same for `apply`.
Return the list of passing classes.

Step D5 — `_stage_tests(report, module_path, mechanic_classes)`: for each mechanic class, compute candidate test paths. Look at the module_path: if `module_path` is under `src/token_world/mechanic/seeds/`, the test lives at `tests/test_mechanic/test_seeds/test_<id>.py` (project tree); if under `<universe>/mechanics/`, the test lives at `<universe>/tests/test_mechanics/test_<id>.py`. Resolve by walking parents: find the first ancestor named `mechanics` or `seeds`, then construct the sibling test path. If test file does not exist → warning `no_tests_found`, continue. If it exists → `subprocess.run([sys.executable, "-m", "pytest", str(test_path), "-q", "--no-header"], capture_output=True, text=True)`; if `returncode != 0` → fail with rule="tests_failed", message=`f"pytest exit {rc}: {stdout[-500:]}"`. NEVER use `shell=True`. The subprocess argv is a fixed list with sanitized path — no injection surface.

Step D6 — `_stage_smoke(report, mechanic_classes)`: Import minimal dependencies lazily to avoid import at module load:
```python
from token_world.graph import KnowledgeGraph
from token_world.mechanic.context import MechanicContext
```
For each cls: instantiate with `()` — any exception → fail with rule="smoke_instantiation_raised". Then construct `kg = KnowledgeGraph(db_path=None)` (or equivalent minimal constructor — read `graph/knowledge_graph.py` for the actual signature), add two nodes via `kg.add_node("_smoke_actor", node_type="agent")` and `kg.add_node("_smoke_target", node_type="entity")`, build `ctx = MechanicContext(kg, actor="_smoke_actor", target="_smoke_target")`, call `cls().check(ctx)`. Wrap in try/except Exception; on raise → fail with rule="smoke_raised", message=`f"{type(e).__name__}: {e}"`. `CheckResult(passed=False, ...)` is NOT a fail — the mechanic refused, which is valid behavior.

Step B — Add `validate-mechanic` CLI command to `src/token_world/cli.py`:
```python
from pathlib import Path
import json as _json
from token_world.mechanic.validation import validate

@cli.command("validate-mechanic")
@click.argument("universe_or_path")
@click.argument("mechanic_id", required=False, default=None)
@click.option("--format", "fmt", type=click.Choice(["human", "json"]), default="human")
def validate_mechanic(universe_or_path: str, mechanic_id: str | None, fmt: str) -> None:
    """Validate a mechanic module and print a structured report.

    Usage:
      token-world validate-mechanic <universe-slug> <mechanic-id>
      token-world validate-mechanic --path <path-to-module.py>

    Exit code 0 on pass, 1 on fail.
    """
    p = Path(universe_or_path)
    if p.is_file() and p.suffix == ".py":
        module_path = p
    else:
        # Resolve via UniverseManager
        from token_world.universe.manager import UniverseManager
        if mechanic_id is None:
            click.echo("Error: mechanic-id is required when universe-or-path is a slug", err=True)
            raise SystemExit(2)
        manager = UniverseManager()
        try:
            universe_dir = manager.load(universe_or_path)
        except (FileNotFoundError, ValueError) as e:
            click.echo(f"Error: {e}", err=True)
            raise SystemExit(2) from e
        module_path = universe_dir / "mechanics" / f"{mechanic_id}.py"
        if not module_path.is_file():
            click.echo(f"Error: mechanic file not found: {module_path}", err=True)
            raise SystemExit(2)

    report = validate(module_path)

    if fmt == "json":
        click.echo(_json.dumps(report.to_dict(), indent=2, sort_keys=True))
    else:
        # human
        status = "PASS" if report.passed else "FAIL"
        click.echo(f"{status} {module_path}")
        for f in report.findings:
            loc = f":{f.line}:{f.col}" if f.line is not None else ""
            click.echo(f"  [{f.severity}] [{f.stage}:{f.rule}] {f.path}{loc} — {f.message}")

    raise SystemExit(0 if report.passed else 1)
```
