For `tell`:
```python
class TellMechanic(Mechanic):
    id = "tell"
    description = "Agent tells another agent a claim about a third entity"
    voluntary = True
    tags: list[str] = ["social", "belief"]

    def check(self, ctx):
        if not ctx.has_node(ctx.actor) or not ctx.has_node(ctx.target):
            return CheckResult(passed=False, reasons=["actor or recipient missing"])
        utt = ctx.query_node(ctx.actor).get("utterance")
        if not isinstance(utt, dict) or not {"about", "property", "value"} <= utt.keys():
            return CheckResult(passed=False, reasons=["actor has no utterance with required shape"])
        return CheckResult(passed=True)

    def apply(self, ctx):
        utt = ctx.query_node(ctx.actor)["utterance"]
        beliefs = dict(ctx.query_node(ctx.target).get("beliefs") or {})
        claim = dict(beliefs.get(utt["about"]) or {})
        claim[utt["property"]] = utt["value"]
        beliefs[utt["about"]] = claim
        return [ctx.mutate(ctx.target, "beliefs", beliefs)]
```

Step D — Update the integration harness (`tests/test_integration/test_use_cases.py`) to READ `blocked_by` on matched mechanics. Logic: at collection time, after loading the seed registry, build a map `{mechanic_id: blocked_by_gap_id}` for all mechanics with a truthy `blocked_by` class attr. If a UC's action verb classified.verb corresponds to a mechanic with `blocked_by`, override the UC's expected_outcome to "blocked" with reason f"stub: blocked by {blocked_by}". This means UC-O02 (persuade) and UC-O06 (cooperate) will now emit skip-reason mentioning the actual gap ID, even if the manifest's expected_outcome field is slightly stale.

Pseudocode for the harness extension:
```python
# At module load: build blocked_by map from seed registry
from token_world.mechanic import MechanicRegistry
_seeds_registry = MechanicRegistry(SEEDS_DIR)
_BLOCKED_BY_MAP = {
    info.id: getattr(_seeds_registry.get_class(info.id), "blocked_by", None)
    for info in _seeds_registry.list_mechanics()
    if getattr(_seeds_registry.get_class(info.id), "blocked_by", None)
}

# In _classify_outcome: if any action's verb maps to a stub, override outcome to "blocked"
def _classify_outcome(fm: dict) -> tuple[str, str | None]:
    for action in fm.get("actions") or []:
        verb = (action.get("classified") or {}).get("verb")
        if verb in _BLOCKED_BY_MAP:
            return "blocked", f"stub: blocked by {_BLOCKED_BY_MAP[verb]}"
    # fall through to existing logic
    outcome = fm.get("expected_outcome") or "pass"
    ...
```

Use `MechanicRegistry.get_class(mechanic_id)` (added in integrate gate — see commit 06fc528). Returns `type[Mechanic]` so class attributes (`blocked_by`, `tags`) are directly readable. Never reach into `registry._classes`.
