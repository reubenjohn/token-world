<objective>
Build the Sonnet-backed observation synthesiser that closes Gap 2 of 05-VERIFICATION.md.
The VisibilityProjector (Plan 05-04) produces a projected state dict; the ChainExecutionEngine
(Phase 2) produces an ExecutionTrace; the Decider (Plan 05-03) may return a RefuseDecision
with a pre-rendered narrative. This plan adds the missing consumer that turns those three
inputs into the grounded observation text the resident agent ultimately reads.

Per D-15 the observer operates under a hard grounding constraint: the system prompt
explicitly forbids referencing facts not present in the projection dict. Phase 5 ships the
constraint plus a cheap substring-based grounding assertion in tests; the full LLM-verifier
rubric is deferred to Phase 6 TEST-04 (per D-15 and pitfall #6).

This plan does NOT wire the observer into a tick pipeline — that is Plan 05-08's job.
Observer here is a self-contained, testable component the orchestrator will call.
</objective>

Create `Observer` dataclass at `src/token_world/engine/observer.py`:

```python
@dataclass(slots=True)
class Observer:
    """Sonnet-backed observation synthesiser (D-15 hard-grounding).

    Consumes a VisibilityProjector projection + ExecutionTrace and produces a
    grounded prose observation the resident agent reads. Refusal narratives
    pass through unchanged.
    """

    client: Any  # anthropic.Anthropic | test fake with .messages.create
    model: str = _MODEL
    max_tokens: int = 1024
    last_input_tokens: int = 0  # populated after each synthesize() for tick-summary cost
    last_output_tokens: int = 0

    def synthesize(
        self,
        *,
        projection: dict[str, dict[str, Any]],
        trace: ExecutionTrace | None,
        refusal_narrative: str | None = None,
        actor_id: str,
        action_text: str = "",
        tick_diag_ctx: Any = None,
    ) -> str:
        """Produce grounded observation text.

        Refusal short-circuit: if refusal_narrative is provided, return it
        verbatim (no LLM call, no rewriting, no fabrication). Diagnostics
        still write so the operator can see the refusal payload.

        Empty-projection fallback: if projection is empty or contains only
        the actor entry with no edges/properties beyond identity, return
        "You can sense nothing — only darkness and silence." (no LLM call).

        Otherwise: build a grounded prompt from (system + projection json +
        trace summary + action_text), call Sonnet, capture token usage on
        self.last_input_tokens / self.last_output_tokens, write diagnostics
        if tick_diag_ctx provided, and return the response text.
        """
```

**System prompt (D-15 hard grounding) — module-level constant `_SYSTEM_PROMPT`:**

```
You are the observation synthesiser for a text-based simulation. Given a
projected state dict and an execution trace, write a short (1-3 sentence)
second-person prose observation describing what the resident agent now
perceives.

HARD GROUNDING CONSTRAINT: use only facts that appear in the provided state
dict. If a property is not present, do not mention it. Do not invent objects,
properties, or sensory details. If the projection is sparse (dark room, no
items), describe only the absence — never invent atmosphere.

Output the observation text directly. No prefix, no suffix, no JSON wrapping.
```

The literal phrase **"use only facts that appear in the provided state"** must appear in
the system prompt — a test asserts substring presence (not full text equality, allowing the
researcher to wordsmith per D-26).

**User prompt template (D-26 discretion on phrasing, must include these data slots):**

```
Actor: {actor_id}
Action attempted: {action_text!r}
Projected state (JSON): {json.dumps(projection, sort_keys=True)}
Execution outcome:
  - Mechanic: {trace.root.mechanic_id} (executed {trace.total_mechanics_executed} mechanics, depth {trace.max_depth_reached})
  - Mutations: {len(_flatten_mutations(trace))} graph changes
  - Truncated: {trace.truncated}
{_render_chain_truncation_note(trace) if trace.truncated else ""}
```

**Empty-projection fallback (test-driven, no LLM call):**

```python
def _is_empty_projection(self, projection: dict) -> bool:
    if not projection:
        return True
    # Single-actor projection with no other nodes and no edges → effectively empty
    if len(projection) == 1:
        only_entry = next(iter(projection.values()))
        if not only_entry.get("edges") and len(only_entry.get("properties", {})) <= 2:
            return True
    return False

# Usage:
if self._is_empty_projection(projection):
    fallback = "You can sense nothing — only darkness and silence."
    if tick_diag_ctx is not None:
        tick_diag_ctx.write_observation(
            prompt="(empty-projection fallback — no LLM call)",
            response=fallback,
            parsed={"text": fallback, "source": "empty_projection_fallback"},
        )
    return fallback
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-OBS-HALLUCINATION | Tampering (information integrity) | observer.py system prompt | mitigate | D-15 hard-grounding constraint in system prompt + Phase-5 substring assertion test (test #10). Full LLM-verifier rubric is Phase 6 TEST-04 (acknowledged limitation). |
| T-05-OBS-PROMPT-INJECTION | Tampering | projection JSON serialised into user prompt | accept | Projection contents are derived from the engine-controlled KnowledgeGraph via VisibilityProjector — no resident-agent free text reaches this prompt. | T-05-OBS-EMPTY-CRASH | Denial of Service | empty projection from dark room or missing actor | mitigate | Empty-projection fallback returns darkness narrative without LLM call (test #3, #4). Pitfall #14 closure. |

<risks>
- **Sonnet model ID drift.** Hardcoded `_MODEL = "claude-sonnet-4-5-20250929"` may need an update before Plan 05-08 integration. Acceptable: dataclass field allows per-instance override; Plan 05-08 integration tests will catch a wrong model ID with a 404.
- **Substring grounding test brittleness.** Pitfall #6 acknowledged: "The room is dimly lit" passes "dim" substring even if dim was hallucinated. Test #10 documents this explicitly. Phase 6 TEST-04 supersedes.
- **Mock client pattern.** D-23 says "no mocking the diagnostics sink"; this plan honours that — the diagnostics writes use a real fake `TickDiagnostics` shape, not a `Mock` object that swallows everything. Build a minimal capturing fake in test file.
- **Token-usage type compatibility.** Phase 4 conftest `_Usage` class has `input_tokens=100, output_tokens=20` literals — these are the values asserted in test #9. Adjust if conftest changes.
- **Refusal narrative passthrough.** Pitfall: tempting to "improve" the refusal text by passing through Sonnet for tone. Don't — D-13 already centralised refusal narrative; Sonnet pass would re-introduce tone drift across refusal sources. Test #5 enforces verbatim return.
</risks>
