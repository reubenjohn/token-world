    **Add to `src/token_world/graph/persistence.py` (GraphPersistence class):**
    - `save_snapshot(self, graph: nx.DiGraph, tick_id: int, summary: str) -> int`:
      - Serialize graph to JSON via `json_graph.node_link_data(graph)` then `json.dumps(..., ensure_ascii=False, allow_nan=False)`
      - Insert into `graph_snapshots` table: tick_id, summary, graph_json, node_count=`graph.number_of_nodes()`, edge_count=`graph.number_of_edges()`
      - Return `cursor.lastrowid` as snapshot_id
    - `load_snapshot(self, snapshot_id: int) -> tuple[nx.DiGraph, int]`:
      - Query `graph_snapshots WHERE snapshot_id = ?`
      - Deserialize: `json_graph.node_link_graph(json.loads(row["graph_json"]), directed=True, multigraph=False)` — CRITICAL: always `directed=True` per Pitfall 1
      - Return (graph, tick_id)
    - `list_snapshots(self) -> list[SnapshotInfo]`:
      - Query all from graph_snapshots ordered by tick_id ASC
      - Return list of SnapshotInfo dataclass instances
    - `prune_snapshots(self, max_count: int = 50) -> list[int]`:
      - If count of snapshots > max_count, delete oldest (by snapshot_id) until count == max_count
      - Return list of deleted snapshot_ids
    - `delete_events_before(self, tick_id: int) -> int`:
      - Delete from graph_events where tick_id < given tick_id
      - Return number of deleted rows

    **Add to `src/token_world/graph/knowledge_graph.py` (KnowledgeGraph class):**
    - `snapshot(self, tick_id: int, summary: str = "") -> int`:
      - Set `self._current_tick = tick_id`
      - If persistence: save current graph state first (`self.save()`), then call `self._persistence.save_snapshot(self._graph, tick_id, summary)`
      - Prune if over retention: `self._persistence.prune_snapshots(max_count=50)` (D-07 discretion: count-based, 50 max)
      - Compact events: determine oldest retained snapshot tick_id, call `self._events.clear_before(oldest_tick)` and `self._persistence.delete_events_before(oldest_tick)`
      - Return snapshot_id
      - If no persistence: raise `RuntimeError("Cannot snapshot without persistence (db_path required)")`
    - `restore(self, snapshot_id: int) -> None`:
      - If no persistence: raise `RuntimeError`
      - Call `graph, tick_id = self._persistence.load_snapshot(snapshot_id)`
      - Replace `self._graph = graph`
      - Set `self._current_tick = tick_id`
      - Save restored state: `self.save()`
    - `list_snapshots(self) -> list[SnapshotInfo]`:
      - If no persistence: return empty list
      - Delegate to `self._persistence.list_snapshots()`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Snapshot summary input | Summary strings come from tick summaries (potentially LLM-generated content) |
| SQLite snapshot storage | Large JSON blobs stored in SQLite TEXT columns |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Tampering | snapshot summary | accept | Summary is display-only metadata, not executable. No injection risk since it's stored as parameterized SQL TEXT. |
| T-01-07 | Denial of Service | snapshot bloat | mitigate | Count-based retention (max 50 snapshots). prune_snapshots() deletes oldest when limit exceeded. Event compaction removes pre-snapshot events. |
| T-01-08 | Tampering | restore() | mitigate | Snapshot data is serialized by our own save_snapshot() using json_graph.node_link_data(). Always deserialize with directed=True, multigraph=False to prevent graph type confusion. |
</threat_model>
