<objective>
Extend Phase 4's diagnostics substrate with an `operator/` namespace so the harness can persist yield signals, authoring transcripts, validation reports, mechanic diffs, and resume outcomes for every tick. This is the storage layer behind `inspect-yield` and `replay-tick` CLIs (built in 04.1-04), and the feedback surface that lets a developer post-mortem an authoring cycle.

Purpose: D-15 locks the operator diagnostics layout inside Phase 4's per-tick folder. Without a dedicated sink API, the harness (04.1-03) would inline filesystem writes and duplicate logic. Shipping the namespace first gives 04.1-03 a clean interface and 04.1-04 a clean reader.

Output: `OperatorDiagnosticsContext` context manager that writes the 4 artefact types listed in RESEARCH §7 (yield_signal.json, authoring_attempts.jsonl, validation/attempt_NN.json, mechanic_diff.patch, resume_outcome.json); `DiagnosticsSink.open_operator_session(tick_id)` integration point; a `OperatorDiagnosticsReader` that loads an already-written session back for `replay-tick`.

    ```python
    class OperatorDiagnosticsContext(AbstractContextManager["OperatorDiagnosticsContext"]):
        def __init__(self, universe_path: Path, tick_id: str) -> None:
            self.universe_path = universe_path
            self.tick_id = str(tick_id)
            self.operator_dir = universe_path / "diagnostics" / f"tick_{self.tick_id}" / "operator"
            self.operator_dir.mkdir(parents=True, exist_ok=True)
            (self.operator_dir / _VALIDATION_DIR).mkdir(exist_ok=True)
            self._closed = False

        def __exit__(self, exc_type, exc, tb):
            if self._closed:
                return None
            outcome: dict[str, Any] = {
                "success": False,
                "mechanic_id": None,
                "cost_usd": None,
                "turns": 0,
                "tick_continued": False,
                "error": f"{exc_type.__name__}: {exc}" if exc else "session_not_closed",
            }
            self.close(outcome)
            return None  # do not suppress exceptions

        def write_yield_signal(self, signal: YieldSignal) -> None:
            path = self.operator_dir / _YIELD_FILE
            _atomic_write_text(path, signal.to_json())

        def append_attempt(self, attempt: dict[str, Any]) -> None:
            path = self.operator_dir / _ATTEMPTS_FILE
            # Non-atomic append is acceptable per JSONL convention; crashes
            # during append may produce a partial final line, which the reader
            # tolerates by skipping unparseable lines (see Task 2).
            with path.open("a", encoding="utf-8") as f:
                f.write(json.dumps(attempt, sort_keys=True) + "\n")

        def write_validation_report(self, attempt_n: int, report: dict[str, Any]) -> None:
            path = self.operator_dir / _VALIDATION_DIR / f"attempt_{attempt_n:02d}.json"
            _atomic_write_json(path, report)

        def write_mechanic_diff(self, diff: str) -> None:
            path = self.operator_dir / _DIFF_FILE
            _atomic_write_text(path, diff)

        def close(self, outcome: dict[str, Any]) -> None:
            merged = {"schema_version": OPERATOR_SCHEMA_VERSION, **outcome}
            _atomic_write_json(self.operator_dir / _OUTCOME_FILE, merged)
            self._closed = True
    ```

    Implement `_atomic_write_text` and `_atomic_write_json` as module-private helpers using `tempfile.NamedTemporaryFile(dir=self.operator_dir, delete=False)` + `os.replace` + fsync.

    ```python
    class OperatorDiagnosticsReader:
        def __init__(self, universe_path: Path, tick_id: str | int) -> None:
            self.universe_path = universe_path
            self.tick_id = str(tick_id)
            self.operator_dir = universe_path / "diagnostics" / f"tick_{self.tick_id}" / "operator"

        def yield_signal(self) -> YieldSignal:
            path = self.operator_dir / _YIELD_FILE
            if not path.exists():
                raise FileNotFoundError(
                    f"No yield signal at {path}. "
                    f"Operator session may not have started for tick {self.tick_id}."
                )
            return YieldSignal.from_json(path.read_text(encoding="utf-8"))

        def attempts(self) -> list[dict[str, Any]]:
            path = self.operator_dir / _ATTEMPTS_FILE
            if not path.exists():
                return []
            out: list[dict[str, Any]] = []
            for line_num, line in enumerate(path.read_text(encoding="utf-8").splitlines(), 1):
                line = line.strip()
                if not line:
                    continue
                try:
                    out.append(json.loads(line))
                except json.JSONDecodeError:
                    # Tolerate the last line being partial (crash mid-append).
                    # Log at debug via logger; upstream reader can show a banner.
                    _logger.warning(
                        "Skipping malformed JSONL line %d in %s", line_num, path,
                    )
            return out

        def validation_reports(self) -> list[dict[str, Any]]:
            vdir = self.operator_dir / _VALIDATION_DIR
            if not vdir.is_dir():
                return []
            return [
                json.loads(p.read_text(encoding="utf-8"))
                for p in sorted(vdir.glob("attempt_*.json"))
            ]

        def mechanic_diff(self) -> str | None:
            path = self.operator_dir / _DIFF_FILE
            if not path.exists():
                return None
            return path.read_text(encoding="utf-8")

        def resume_outcome(self) -> dict[str, Any] | None:
            path = self.operator_dir / _OUTCOME_FILE
            if not path.exists():
                return None
            return json.loads(path.read_text(encoding="utf-8"))

        @property
        def schema_version(self) -> int:
            outcome = self.resume_outcome()
            if outcome is None:
                return OPERATOR_SCHEMA_VERSION  # Assume current if unclosed
            version = outcome.get("schema_version", 1)
            if version != OPERATOR_SCHEMA_VERSION:
                raise ValueError(
                    f"Unsupported operator diagnostics schema_version={version} at "
                    f"{self.operator_dir / _OUTCOME_FILE}; "
                    f"this build understands v{OPERATOR_SCHEMA_VERSION}."
                )
            return version
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Disk (historical diagnostics) → `OperatorDiagnosticsReader` | Session files may be edited by users, stale from older builds, or corrupted mid-write. |
| Operator harness (04.1-03) → `OperatorDiagnosticsContext` | Harness passes authoring-attempt payloads derived from SDK messages; may contain arbitrary strings including prompt-injection-crafted content. Stored verbatim for audit. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04.1-05 | Tampering | `OperatorDiagnosticsReader.schema_version` reading tampered resume_outcome.json | mitigate | Unknown `schema_version` raises `ValueError` (Task 2 test). Prevents silent acceptance of an incompatible-shape outcome payload that could cause `replay-tick` to misrender. |
| T-04.1-06 | Tampering | Partial / corrupted JSONL attempt lines | mitigate | `attempts()` skips unparseable lines with a logged warning rather than crashing; guards against the common failure mode of a crashed harness leaving a truncated last line. Audited via Task 2 test. |
| T-04.1-07 | Information Disclosure | Prompt-injection content stored verbatim in `authoring_attempts.jsonl` | accept | Diagnostics are developer-local; the universe folder is explicitly scoped as operator-trusted storage per Phase 4 D-21. Content is displayed by `replay-tick` (04.1-04) — rendering does no shell interpolation (Task 2 in 04.1-04 will use Click's `echo` not template rendering). |
| T-04.1-08 | Denial of Service | Unbounded `authoring_attempts.jsonl` growth | accept | Per-tick file is bounded by `max_turns=20` × message size (~few KB each). Cross-tick cleanup is Phase 4's `prune-diagnostics` CLI's job. Revisit if Phase 6's N-turn runner changes the calculus. |
| T-04.1-09 | Tampering | Non-atomic append to JSONL | accept | Atomic-append semantics aren't achievable for JSONL without rewrites; mitigated by reader's lenient parsing (T-04.1-06). Critical artefacts (yield_signal, validation reports, resume_outcome) use atomic `os.replace` writes. |
