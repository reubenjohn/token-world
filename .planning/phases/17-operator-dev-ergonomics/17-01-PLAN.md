   - Computes `diag_base = universe_dir / "diagnostics" / f"tick_{tick_id}"`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → file paths | --stage and tick_id are operator-controlled and used to construct filesystem paths |
| universe_dir / git subprocess | universe_dir is user-supplied; git is run as subprocess inside it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01-01 | Spoofing | load_stage_data path construction | mitigate | tick_id is used directly in path; validate it is alphanumeric/underscore only before constructing path |
| T-17-01-02 | Info Disclosure | --raw flag exposes full LLM prompts | accept | operator-only CLI tool; prompts are already on disk; no additional exposure |
| T-17-01-03 | Tampering | git subprocess in _git_first_commit | mitigate | run with timeout=5; capture_output=True; never shell=True; cwd is universe_dir (verified to exist by caller) |
| T-17-01-04 | DoS | git log on very large history | mitigate | 5s timeout on subprocess.run; graceful None return on timeout |
</threat_model>
