# Phase 07.1 Plan 01: llm_backend.py — LLMBackend Protocol + AnthropicSDKBackend + ClaudeCLIBackend + get_backend factory Summary

One-liner: Pluggable LLM backend abstraction — single `LLMBackend` Protocol with two implementations (`AnthropicSDKBackend`, `ClaudeCLIBackend`) selected by `TOKEN_WORLD_BACKEND` env var; unblocks Plan 02's Classifier/Observer/ResidentAgent refactor and Phase 6 zero-cost live UAT.

## What Shipped

Five new public symbols (four exported from `token_world.engine`, one module-internal helper):

| Symbol                     | Role                                                                                                                    |
| -------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| `LLMBackend` (Protocol)    | Single-method interface: `call(*, model: str, system: str, prompt: str, max_tokens: int) -> str`. D-01.                 |
| `AnthropicSDKBackend`      | Wraps injected `anthropic.Anthropic` client. Returns `""` on empty content. **Does NOT** strip markdown fences (D-03).  |
| `ClaudeCLIBackend`         | `subprocess.run(["claude", "--model", <model>, "-p", "<system>\\n\\n<prompt>"], timeout=120, check=True, capture_output=True, text=True)`. Strips markdown fences via `_strip_markdown_fences` (D-03). |
| `get_backend()`            | Factory reading `TOKEN_WORLD_BACKEND`; `.strip().lower() == "claude-cli"` → `ClaudeCLIBackend()`, else lazy-imports `anthropic.Anthropic` and returns `AnthropicSDKBackend(Anthropic())`. D-08. |
| `_strip_markdown_fences()` | Module-internal helper. Idempotent; handles `` ```json ``, plain `` ``` ``, unfenced, whitespace-padded, missing-close, empty. D-03. |

### The most subtle decision: D-03 — fence stripping is CLI-path only

The Anthropic SDK returns unfenced plain text. The `claude -p` CLI subprocess wraps JSON output in `` ```json\n...\n``` `` fences even when the prompt explicitly forbids it (validated 2026-04-14 live probe). Placing fence stripping inside `ClaudeCLIBackend.call()` (and **only** there) keeps the adapter boundary clean — every downstream caller (classifier, observer, resident agent) sees a unified fence-free string regardless of which backend is active. The SDK backend **must not** strip because real SDK responses containing literal triple-backticks would be mangled.

This is captured in two ways in the test suite:
- `TestAnthropicSDKBackend.test_does_not_strip_markdown_fences` asserts the SDK backend preserves fenced input verbatim.
- `TestClaudeCLIBackend.test_strips_markdown_fences_from_subprocess_output` asserts the CLI backend unwraps `` ```json\n{...}\n``` `` to `{...}`.

### Lazy `from anthropic import Anthropic` inside `get_backend`

The `anthropic` import is done inside the function body, not at module top. This preserves the Phase 5 D-01 "no module-load Anthropic instantiation" pattern — tests patch `anthropic.Anthropic` before calling `get_backend()` and the patch is honoured, environments without the SDK installed can still `import token_world.engine.llm_backend`, and `TOKEN_WORLD_BACKEND=claude-cli` paths never touch the SDK at all.

- **Why this doesn't change plan semantics:** the done criteria for Task 1 and Task 2 are about test-coverage staging, not module-shape staging. Plan 02 consumes the full public API regardless of commit layout. Three commits landed (one per task) as required.
