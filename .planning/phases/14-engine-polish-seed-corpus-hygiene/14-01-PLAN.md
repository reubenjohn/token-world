<objective>
Fix ENGINE-05: a refused tick's observation may contain "You try, but" twice because
a mechanic's check() reason string itself starts with "You try, but …" and engine.py
wraps it again via RefusalTemplate.render("mechanic_check_failed", {"reason": ...}).

Template for mechanic_check_failed is: `"You try, but {reason}."` — so if {reason}
is "You try, but the knife is too dull", the output is
"You try, but You try, but the knife is too dull." — double-wrapped.

Fix: make RefusalTemplate.render() strip any leading "You try, but " prefix from the
`reason` kwarg before templating, or more robustly, make the whole render() idempotent:
if the already-rendered string starts with a known wrapper prefix, return it as-is.

Purpose: SC-1 truthfulness — the resident agent never reads a grammatically broken
doubled refusal.

Output: regression test + patched refusal.py, confirmed green.
</objective>

    Use a helper `_strip_wrapper(s: str) -> str` that strips "You try, but " prefix repeatedly
    until none remain, then strips trailing " " or "." before returning.

    Implementation approach (idempotent strip):
    ```python
    _WRAPPER_PREFIX = "You try, but "

    def _strip_wrapper(s: str) -> str:
        """Strip repeated 'You try, but ' prefix from a reason string."""
        while s.startswith(_WRAPPER_PREFIX):
            s = s[len(_WRAPPER_PREFIX):]
        # Also strip leading period/space left from prior formatting
        s = s.lstrip(". ")
        return s
    ```
