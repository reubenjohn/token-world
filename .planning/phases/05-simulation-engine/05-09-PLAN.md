<objective>
Close Gap 5 of 05-VERIFICATION.md by replacing the Phase 0 MCP stub bodies with real
implementations that drive SimulationEngine (Plan 05-08), MechanicRegistry, and
KnowledgeGraph.restore. Per UNIV-03 the MCP surface is exactly three tools —
`resume_tick`, `rollback`, `list_mechanics` — and Claude Code running inside a universe
folder uses these to step the simulation, inspect mechanics, and recover from bad state.

The current implementation (`src/token_world/mcp_server.py:89`) returns
`"This is a Phase 0 stub."` for all three tools. The 05-VERIFICATION anti-pattern table
explicitly flags this as a Blocker; `list_mechanics`, `resume_tick`, `rollback` must now
do real work.

This plan is intentionally thin: the engine, registry, and graph APIs are all already built
by Plans 05-01 .. 05-08 and Phases 1/2/4. The MCP server's job is to translate JSON-RPC
params into those API calls and JSON-RPC responses back, plus defensive input validation
against the small attack surface of untrusted tool params (T-05-MCP-*).

**v1 simplification:** resume_tick in v1 does NOT implement full "resume from halted tick"
state restoration. Per D-02 the registry auto-scans on every run_tick — so if the operator
authored a missing mechanic between yield and resume, the next run_tick naturally picks it
up. resume_tick in v1 is equivalent to "run one tick"; the tick_id increment is handled
inside the engine. Fuller state-restoration semantics (re-execute a specific yielded tick_id
with its original classified_action) are a later enhancement; explicitly out of scope here.

**Security note (per T-00-05 Phase 0 threat):** JSON-RPC stdio is the primary attack
surface. Parse errors already return -32700 (Phase 0), invalid methods -32601 (Phase 0),
invalid params must now return -32602 for missing/invalid tool params. All three tools
wrap handlers in a broad try/except → -32603 Internal error.
</objective>

Define three private tool functions + two helpers at module level:

```python
class _InvalidParams(ValueError):
    """Raised by tool functions on missing/invalid JSON-RPC params."""


def _jsonrpc_error(req_id: Any, code: int, message: str) -> dict:
    return {"jsonrpc": "2.0", "id": req_id, "error": {"code": code, "message": message}}


def _require_universe_path(arguments: dict) -> Path:
    """Extract + validate universe_path from tool arguments.

    Returns the resolved Path. Raises _InvalidParams on missing or invalid input.
    Path-traversal defence: rejects '..' segments in the raw input.
    """
    raw = arguments.get("universe_path")
    if not isinstance(raw, str) or not raw:
        raise _InvalidParams("missing or non-string 'universe_path'")
    if ".." in Path(raw).parts:
        raise _InvalidParams("'universe_path' contains disallowed '..' segment")
    path = Path(raw).resolve()
    if not path.is_dir():
        raise _InvalidParams(f"universe_path does not exist or is not a directory: {path}")
    return path


_anthropic_factory: Any = None  # module-level; tests monkeypatch to inject MockAnthropicClient


def _build_anthropic_client() -> Any:
    """Return an Anthropic client. Tests monkeypatch _anthropic_factory."""
    if _anthropic_factory is not None:
        return _anthropic_factory()
    # Lazy import so the MCP module doesn't crash if anthropic is missing at import time
    from anthropic import Anthropic
    return Anthropic()


def _tool_resume_tick(arguments: dict) -> dict:
    """Run one simulation tick and return a result payload.

    Required arguments: universe_path, action_text, actor.
    """
    universe_path = _require_universe_path(arguments)
    action_text = arguments.get("action_text")
    actor = arguments.get("actor")
    if not isinstance(action_text, str) or not action_text.strip():
        raise _InvalidParams("missing or empty 'action_text'")
    if not isinstance(actor, str) or not actor.strip():
        raise _InvalidParams("missing or empty 'actor'")

    # Lazy imports — keep module import light so tests that don't exercise resume_tick
    # don't pay for the engine + anthropic import chain.
    from token_world.engine import SimulationEngine
    from token_world.graph import KnowledgeGraph

    db_path = universe_path / "universe.db"
    graph = KnowledgeGraph(db_path=db_path)
    if db_path.exists():
        graph.load()

    engine = SimulationEngine(
        universe_path=universe_path,
        graph=graph,
        anthropic_client=_build_anthropic_client(),
    )
    result = engine.run_tick(action_text, actor)

    # Persist graph state after tick
    graph.save()

    payload: dict[str, Any] = {
        "tick_id": result.tick_id,
        "kind": result.kind,
        "observation": result.observation,
        "refusal_reason": result.refusal_reason,
    }
    if result.yield_signal is not None:
        payload["yield_signal"] = json.loads(result.yield_signal.to_json())
    return payload


def _tool_list_mechanics(arguments: dict) -> dict:
    """List all registered mechanics.

    Required arguments: universe_path.
    Optional: filter (substring match against mechanic id, case-insensitive).
    """
    universe_path = _require_universe_path(arguments)
    filter_str = arguments.get("filter")

    from token_world.mechanic.registry import MechanicRegistry

    registry = MechanicRegistry(universe_path / "mechanics", universe_dir=universe_path)
    mechanics = registry.list_mechanics()

    if isinstance(filter_str, str) and filter_str:
        needle = filter_str.lower()
        mechanics = [m for m in mechanics if needle in m.id.lower()]

    return {
        "count": len(mechanics),
        "mechanics": [
            {
                "id": m.id,
                "description": m.description,
                "voluntary": m.voluntary,
                "tags": list(m.tags),
            }
            for m in mechanics
        ],
    }


def _tool_rollback(arguments: dict) -> dict:
    """Restore the graph to a previous snapshot.

    Required arguments: universe_path, snapshot_id (int).
    """
    universe_path = _require_universe_path(arguments)
    raw_snapshot = arguments.get("snapshot_id")
    try:
        snapshot_id = int(raw_snapshot)
    except (TypeError, ValueError) as exc:
        raise _InvalidParams(f"'snapshot_id' must be an integer: got {raw_snapshot!r}") from exc

    from token_world.graph import KnowledgeGraph

    db_path = universe_path / "universe.db"
    if not db_path.exists():
        raise _InvalidParams(f"no universe.db at {db_path}")

    graph = KnowledgeGraph(db_path=db_path)
    graph.load()
    pre_restore_tick = graph.current_tick
    graph.restore(snapshot_id)
    graph.save()

    return {
        "ok": True,
        "snapshot_id": snapshot_id,
        "restored_to_tick": graph.current_tick,
        "rolled_back_from_tick": pre_restore_tick,
    }
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JSON-RPC stdin → mcp_server dispatcher | Untrusted LLM-harness input crosses into the server |
| tool arguments dict → tool implementation | Per-tool untrusted params (universe_path, action_text, snapshot_id) |
| universe_path → filesystem | Path interpolation to access db, mechanics, config |
| SimulationEngine raises → JSON-RPC error | Exception path must not leak stack traces over stdout |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-MCP-JSON-INJECTION | Tampering | action_text or actor fields interpolated into prompts | accept | Classifier (Plan 05-01) already validates action_text via structured output and min_confidence; actor must match a known_node_id or classifier returns no_such_target. This plan adds basic type/emptiness checks only. |
| T-05-MCP-PATH-TRAVERSAL | Tampering | universe_path could contain '..' to read arbitrary filesystem | mitigate | `_require_universe_path` rejects paths containing '..' in Path(raw).parts before resolving. Test #7. |
| T-05-MCP-ROLLBACK-WIPE | Denial of Service | malicious snapshot_id=0 or negative → graph.restore may corrupt state | accept | Phase 1's KnowledgeGraph.restore raises ValueError on unknown snapshot_id (per persistence.py:229). The catch-all in tools/call converts to -32603. Documented — DoS by intentional rollback is an authorised-operator concern, not an unauthenticated-caller concern (MCP is local stdio only). |
| T-05-MCP-STACK-TRACE-LEAK | Information Disclosure | Python traceback leaked in JSON-RPC error.message reveals filesystem layout | mitigate | exception handler writes full trace to `sys.stderr`, only `str(exc)` goes to the JSON response. Test #21. |
| T-05-MCP-STATELESS-RACE | Tampering | Two concurrent MCP clients → double-tick on shared universe.db | accept | SQLite handles concurrent writes via locks; ticks may serialise. Hobby-scale, local stdio — multiple concurrent MCP clients is not a Phase 5 scenario. |
</threat_model>

<risks>
- **Lazy import of SimulationEngine / KnowledgeGraph keeps module-level import cheap.** Alternative: import at top of file. Chosen lazy so `tests/test_mcp_server.py` Phase 0 tests (if any) don't pay for the engine import chain when only testing the stub dispatcher. Acceptable minor performance cost on the first tool/call.
- **`_anthropic_factory` monkeypatch surface.** Tests set `token_world.mcp_server._anthropic_factory = lambda: MockAnthropicClient([...])`. Document this pattern in the module docstring so future test authors find it. Alternative: constructor injection (not possible since mcp_server is a module, not a class).
