<objective>
Build the per-tick TickSummary writer that closes Gap 4 of 05-VERIFICATION.md. Per D-20 the
engine must write `universe/tick_summaries/ticks/tick_<id>.json` after every `run_tick` so
Phase 6 SIM-12 (batch + epoch compression) and operator-side tooling (CLI replay, gap
analysis) have a structured per-tick record.

The Pydantic `TickSummary` schema is **already defined** in `src/token_world/engine/models.py:118`
(per Plan 05-01 SUMMARY). This plan adds:
1. The `TickSummaryWriter` that serialises a `TickSummary` instance to disk (atomic write,
   idempotent path-by-tick-id, mkdir parents).
2. The `build_tick_summary(...)` factory that converts orchestrator-raw inputs (verdict,
   decision, trace, mutations, observation, timing, token usage) into a `TickSummary`
   instance — the orchestrator (Plan 05-08) calls this once per tick.
3. Per-stage cost computation from Haiku/Sonnet token rates, declared as module constants
   so cost-per-tick is auditable in source.

This plan does NOT wire the writer into the tick pipeline — that is Plan 05-08's job.
The writer is also distinct from the diagnostics `summary.json` (which lives in
`diagnostics/tick_<id>/summary.json` and is written by `TickDiagnostics.finalize`); per
D-22 they may carry overlapping content but live in different roots.
</objective>

```python
"""TickSummaryWriter — persist per-tick structured summaries (D-20, SIM-11).

Writes universe/tick_summaries/ticks/tick_<id>.json after every run_tick.
The TickSummary Pydantic schema lives in :mod:`token_world.engine.models`
(defined by Plan 05-01); this module adds the writer and the orchestrator-facing
``build_tick_summary`` factory that turns raw stage outputs into a TickSummary.

Forward-compatibility (D-21): schema_version=1 declared in every file. Phase 6
SIM-12 batch compressor consumes these files; the schema is locked.

Cost accounting (D-24): per-stage USD costs computed from token usage and
hardcoded model-rate constants. Operator audits via the JSON files; Phase 5
ships no circuit breakers.
"""

from __future__ import annotations

import json
from dataclasses import dataclass
from datetime import UTC, datetime
from pathlib import Path
from typing import Any

from token_world.engine.models import (
    Decision,
    ExecuteDecision,
    RefuseDecision,
    TickSummary,
    YieldDecision,
)
from token_world.graph import Mutation
from token_world.mechanic.diagnostics import _atomic_write_json
from token_world.mechanic.trace import ExecutionTrace, TraceNode

# Per-million-token USD rates — D-24 visibility, no circuit breaker.
# Adjust in source if Anthropic pricing changes; verify in tests then bump.
_HAIKU_INPUT_PER_MTOK = 1.00
_HAIKU_OUTPUT_PER_MTOK = 5.00
_SONNET_INPUT_PER_MTOK = 3.00
_SONNET_OUTPUT_PER_MTOK = 15.00


def _stage_cost_usd(stage: str, input_tokens: int, output_tokens: int) -> float:
    """Compute USD cost for a stage's token usage."""
    if stage == "classifier":
        in_rate, out_rate = _HAIKU_INPUT_PER_MTOK, _HAIKU_OUTPUT_PER_MTOK
    elif stage == "observer":
        in_rate, out_rate = _SONNET_INPUT_PER_MTOK, _SONNET_OUTPUT_PER_MTOK
    else:
        # Unknown stage: zero cost (forward-compat for future stages)
        return 0.0
    return (input_tokens * in_rate + output_tokens * out_rate) / 1_000_000


def _flatten_trace_mutations(trace: ExecutionTrace | None) -> list[Mutation]:
    """Walk trace tree and collect every Mutation recorded by every TraceNode."""
    if trace is None:
        return []
    result: list[Mutation] = []
    stack: list[TraceNode] = [trace.root]
    while stack:
        node = stack.pop()
        result.extend(node.mutations)
        stack.extend(node.children)
    return result


def _mutations_to_json_list(mutations: list[Mutation]) -> list[list[Any]]:
    """Convert Mutations to D-20 4-tuple form: [target, property, old, new]."""
    return [[m.target, m.property, m.old_value, m.new_value] for m in mutations]


def build_tick_summary(
    *,
    tick_id: str,
    action_text: str,
    decision: Decision,
    classified_action: dict[str, Any] | None,
    trace: ExecutionTrace | None,
    observation_text: str | None,
    duration_ms: int,
    classifier_input_tokens: int = 0,
    classifier_output_tokens: int = 0,
    observer_input_tokens: int = 0,
    observer_output_tokens: int = 0,
) -> TickSummary:
    """Construct a TickSummary from orchestrator-raw stage outputs.

    Switches on Decision.kind:
        - ExecuteDecision: matched_mechanic_id, observation_text, mutations populated
        - YieldDecision:   yielded=True, matched/observation/refusal_reason all None
        - RefuseDecision:  refused=True, refusal_reason=decision.reason_code
    """
    yielded = isinstance(decision, YieldDecision)
    refused = isinstance(decision, RefuseDecision)
    matched_id = decision.mechanic_id if isinstance(decision, ExecuteDecision) else None
    refusal_reason = decision.reason_code if isinstance(decision, RefuseDecision) else None

    raw_mutations = _flatten_trace_mutations(trace)
    mutations_field = {
        "count": len(raw_mutations),
        "list": _mutations_to_json_list(raw_mutations),
    }

    tokens_by_stage = {
        "classifier": {"in": classifier_input_tokens, "out": classifier_output_tokens},
        "observer": {"in": observer_input_tokens, "out": observer_output_tokens},
    }
    cost_by_stage = {
        "classifier": _stage_cost_usd(
            "classifier", classifier_input_tokens, classifier_output_tokens
        ),
        "observer": _stage_cost_usd(
            "observer", observer_input_tokens, observer_output_tokens
        ),
    }

    return TickSummary(
        tick_id=tick_id,
        timestamp_iso=datetime.now(UTC).strftime("%Y-%m-%dT%H:%M:%SZ"),
        action_text=action_text,
        classified_action=classified_action,
        matched_mechanic_id=matched_id,
        yielded=yielded,
        refused=refused,
        refusal_reason=refusal_reason,
        mutations=mutations_field,
        observation_text=observation_text,
        duration_ms=duration_ms,
        llm_tokens_by_stage=tokens_by_stage,
        llm_cost_usd_by_stage=cost_by_stage,
    )


@dataclass(slots=True)
class TickSummaryWriter:
    """Persist a TickSummary to universe/tick_summaries/ticks/tick_<id>.json."""

    def write(self, summary: TickSummary, universe_dir: Path) -> Path:
        """Write the summary; return the resolved file path.

        Idempotent: same tick_id → same path → atomic overwrite. Creates
        tick_summaries/ticks/ if missing (mkdir parents). Uses the Phase 4
        atomic-write helper so partial writes are never observed by readers.
        """
        out_dir = universe_dir / "tick_summaries" / "ticks"
        out_dir.mkdir(parents=True, exist_ok=True)
        out_path = out_dir / f"tick_{summary.tick_id}.json"
        # Pydantic → dict → JSON via _atomic_write_json.
        # model_dump() preserves Literal[1] for schema_version.
        _atomic_write_json(out_path, json.loads(summary.model_dump_json()))
        return out_path
```

| T-05-SUMMARY-PARTIAL-WRITE | Information Disclosure / Tampering | tick_<id>.json read mid-write returns partial JSON | mitigate | Reuse `_atomic_write_json` (tempfile + os.replace) from Phase 4 — same crash-safety guarantee as diagnostics summary.json. Test #15. |
| T-05-SUMMARY-NON-JSON-PROPS | Tampering (data integrity) | mechanic mutations carry non-JSON-serialisable property values (set/tuple/bytes) | accept | KnowledgeGraph already enforces ALLOWED_PROPERTY_TYPES (json-serialisable only) per CLAUDE.md invariant. No mitigation here — this is an invariant-violation upstream. |
| T-05-SUMMARY-PATH-COLLISION | Tampering | hostile tick_id (e.g., "../etc/passwd") could escape tick_summaries/ | accept | tick_id originates from engine internals (not resident agent text). Plan 05-08 generates tick_id internally as a monotonic int per Phase 1 KnowledgeGraph.snapshot() pattern. Defense-in-depth: f-string interpolation prevents shell escapes, and `.json` suffix ensures no executable extension. Documented constraint: orchestrator-supplied tick_ids are trusted. |

<risks>
- **TickSummary field order in written file is alphabetical** because `_atomic_write_json` calls `json.dump(obj, f, sort_keys=True)`. This is the documented behaviour; tests should assert content, not key order.
- **Cost rates may drift.** Documented in module: "Adjust in source if Anthropic pricing changes." Don't gate on exact USD floats in tests — use whole-million-token assertions to keep the math clean (test #11, #12).
- **Pydantic `model_dump_json` vs `model_dump`.** We use `json.loads(summary.model_dump_json())` to get a strict-JSON dict (handles Literal[1] correctly). Plain `model_dump()` returns a Python dict that may contain non-JSON Python types if model fields are extended later. Defensive choice; pydantic v2 handles both, but the json roundtrip is the contract test #17 enforces.
- **Plan 05-08 must call `build_tick_summary` even for refuse/yield paths.** This plan provides the factory that handles all three; the orchestrator must not skip writing on refuse/yield (per D-20 — every tick gets a summary, refused or not).
- **Diagnostics summary.json vs tick_summaries summary.** D-22 says diagnostics gets its own per-tick `summary.json` populated by `TickDiagnostics.set_summary` + `finalize` (Phase 4). This plan's `tick_summaries/ticks/tick_<id>.json` is a SEPARATE file in a SEPARATE root. Don't conflate. Plan 05-08 will populate both.
</risks>
