### 3. Mechanic Class Contract
Show a minimal example with commentary:
```python
from __future__ import annotations
from typing import TYPE_CHECKING
from token_world.graph import Mutation
from token_world.mechanic.protocol import CheckResult, Mechanic
if TYPE_CHECKING:
    from token_world.mechanic.context import MechanicContext

class PickupMechanic(Mechanic):
    """Agent picks up a target entity if within reach and inventory capacity allows."""
    id = "pickup"
    description = "Agent picks up an entity"
    voluntary = True
    tags: list[str] = ["object_interaction"]

    def check(self, ctx: "MechanicContext") -> CheckResult:
        if not ctx.has_node(ctx.actor) or not ctx.has_node(ctx.target):
            return CheckResult(passed=False, reasons=["actor or target missing"])
        return CheckResult(passed=True)

    def apply(self, ctx: "MechanicContext") -> list[Mutation]:
        return [ctx.add_edge(ctx.actor, ctx.target, relation="holds")]
```

### 6. The Validation Gate (D-14)
- Forbidden imports: `networkx`, `networkx.*`, `token_world.graph.knowledge_graph` → with WHY: mechanics must go through `MechanicContext` so all mutations are logged and snapshots stay consistent.
- Forbidden calls: `eval`, `exec`, `__import__`, `compile`, `globals`, bare `open` → with WHY: authoring discipline + no dynamic-code loopholes.
- **CRITICAL note:** "These AST rules are a reasonable-effort pre-runtime control, NOT a sandbox. Dynamic imports via `importlib` or attribute access like `sys.modules['networkx']` WILL bypass them. v1 runs mechanic code directly; snapshots and rollback are the runtime safety net. Runtime sandboxing (RestrictedPython) is a v2 concern." (T-04-AST-BYPASS acknowledgment.)

### 7. The `_helpers.py` Convention (D-05, D-11)
- Underscore-prefixed `.py` files are NOT registered as mechanics but CAN be imported by sibling mechanics.
- Default to free functions; introduce `SpatialMechanic(Mechanic)`-style base classes only when ≥3 mechanics share a pattern (D-11).
- Example: a shared `_find_open_passage(ctx, src, dst)` helper that both `passage_move` and `try_door` import.

### 8. Framework-Gap-Stub Convention (D-38)
Document the `blocked_by` class attribute pattern from RESEARCH.md Pitfall 6. When a mechanic needs a framework extension that has not yet shipped (e.g., MECH12 needs `ctx.actors`, MECH09 needs `llm_adjudicated` category), ship the mechanic as a stub:

class CooperateMechanic(Mechanic):
    id = "cooperate"
    description = "Multi-actor cooperative action (MECH12; blocked on GAP-ENG05)"
    voluntary = True
    tags: list[str] = ["social", "cooperation"]

        return CheckResult(

        )

    def apply(self, ctx: "MechanicContext") -> list[Mutation]:
        return []
```

Explicitly state: the stub MUST NOT import a symbol that does not exist (Pitfall 6) — otherwise validation stage 3 (import) fails. Stubs simulate the gap via a `check() → passed=False` with a standardized reason.

### 9. Reactive-Cycle Cautions (GAP-MECH26 absorbed)
- Involuntary mechanics that mutate properties they also watch can create infinite cycles.
- The engine's `max_chain_depth=10` backstops runaway loops, BUT:
  - Rule of thumb: before adding an involuntary mechanic, write down what mutations trigger it AND what mutations it produces. If those overlap, reconsider.
  - Prefer matchers that watch property transitions (`old != new`) rather than "every set_property of X".
  - Use `ExecutionTrace` in tests to inspect the chain tree and catch unintended re-firing.

### 10. Trust Boundary Rationale (GAP-MECH19 absorbed)
- Under inversion of control, ALL mechanics are operator-authored. There is no `source='llm_generated'` distinction vs `source='human'`; all code goes through the same validation gate.
- No `reviewed=true/false` metadata is introduced. The gate IS the review step; passing validation IS "reviewed".
- This resolves GAP-MECH19 without additional code.

### 11. Testing Conventions
- Tests use `GraphBuilder` from `tests/test_graph/conftest.py` (framework) or its universe equivalent.
- Pattern: build a minimal graph, instantiate the mechanic, call `check(ctx)` and `apply(ctx)`, assert mutations.
- Multi-mechanic chain tests use `ChainExecutionEngine`.

### 12. Common Anti-Patterns
Bullet list with 1-line descriptions:
- Direct `kg._graph.add_node(...)` (violates mutation-mediated access)
- Mutable class attributes as containers (e.g., `queue: list = []`) — use instance state inside `check`/`apply`
- `assert` statements for validation (users can run Python with `-O`; use `CheckResult(passed=False, ...)`)
- Storing non-JSON-serializable values in graph props (set, tuple, bytes, custom objects) — enforced by `ALLOWED_PROPERTY_TYPES`
- Hardcoded node IDs (use `kg.claim_id("name")` for new nodes)
- Mechanics that block waiting for external IO (HTTP, filesystem) — not supported by the execution model

from __future__ import annotations

from typing import TYPE_CHECKING

from token_world.graph import Mutation
from token_world.mechanic.protocol import CheckResult, Mechanic

if TYPE_CHECKING:
    from token_world.mechanic.context import MechanicContext


class {class_name}(Mechanic):
    """{description}.

    Preconditions:
        - TODO
    Side effects:
        - TODO
    """

    id = "{mechanic_id}"
    description = "{description}"
    voluntary = {voluntary}
    tags: list[str] = []

    def check(self, ctx: "MechanicContext") -> CheckResult:
        # TODO: implement preconditions
        return CheckResult(passed=False, reasons=["TODO: implement check"])

    def apply(self, ctx: "MechanicContext") -> list[Mutation]:
        # TODO: implement side effects
        return []
'''

_TEST_SKELETON = '''"""Tests for the ``{mechanic_id}`` mechanic."""

from __future__ import annotations

import pytest


@pytest.mark.skip(reason="TODO: implement test for {mechanic_id} mechanic")
def test_{mechanic_id}_placeholder() -> None:
    """Placeholder — fill in after implementing check()/apply()."""
    assert False, "TODO"
'''


def _camel_case(s: str) -> str:
    return "".join(part.capitalize() or "_" for part in s.split("_"))


@cli.command("scaffold-mechanic")
@click.argument("universe_slug")
@click.option("--id", "mechanic_id", required=True, help="Unique mechanic id (lowercase_with_underscores)")
@click.option("--voluntary/--involuntary", default=True, help="Voluntary (default) or involuntary")
@click.option("--description", default="TODO: describe this mechanic", help="One-line description")
def scaffold_mechanic(universe_slug: str, mechanic_id: str, voluntary: bool, description: str) -> None:
    """Scaffold a new mechanic in a universe (module + test stub)."""
    import re as _re
    if not _re.match(r"^[a-z][a-z0-9_]*$", mechanic_id):
        click.echo("Error: mechanic-id must be lowercase_with_underscores", err=True)
        raise SystemExit(2)
    from token_world.universe.manager import UniverseManager
    try:
        universe_dir = UniverseManager().load(universe_slug)
    except (FileNotFoundError, ValueError) as e:
        click.echo(f"Error: {e}", err=True)
        raise SystemExit(1) from e
    module_path = universe_dir / "mechanics" / f"{mechanic_id}.py"
    test_path = universe_dir / "tests" / "test_mechanics" / f"test_{mechanic_id}.py"
    if module_path.exists():
        click.echo(f"Error: {module_path} already exists", err=True)
        raise SystemExit(1)
    if test_path.exists():
        click.echo(f"Error: {test_path} already exists", err=True)
        raise SystemExit(1)
    class_name = _camel_case(mechanic_id) + "Mechanic"
    module_path.parent.mkdir(parents=True, exist_ok=True)
    test_path.parent.mkdir(parents=True, exist_ok=True)
    module_path.write_text(_MECHANIC_SKELETON.format(
        class_name=class_name, mechanic_id=mechanic_id,
        description=description, voluntary=voluntary,
    ), encoding="utf-8")
    test_path.write_text(_TEST_SKELETON.format(mechanic_id=mechanic_id), encoding="utf-8")
    click.echo(f"Scaffolded {module_path}")
    click.echo(f"Scaffolded {test_path}")
```
