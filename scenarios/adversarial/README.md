# Adversarial Scenario Pack

This directory contains five canonical adversarial scenarios for the Token World
playtest runner. Each scenario is a YAML file that can be passed to
`token-world playtest <slug> --scenario scenarios/adversarial/<name>.yaml`.

All scenarios use deterministic seeds in the range [100, 110] for reproducibility
across CI and local runs.

---

## nonsense_barrage

**File:** `nonsense_barrage.yaml`
**Turns:** 20 | **Seed:** 100 | **All inject:nonsense**

Fires 20 nonsense strings (random gibberish generated by `InjectionSampler`) at the
engine in sequence. The primary stress target is the classifier's `no_viable_action`
path — the classifier must cleanly refuse or return a low-confidence verdict rather
than hallucinating a plausible mechanic.

**Primary D-12 metric:** `mechanic_match_rate` (should be low, ~0.0–0.2 on a healthy engine).
**Expected profile:** High refusal rate; `observation_groundedness` near 0 (observer doesn't
run on refused/no-viable paths); `action_novelty` high (every nonsense string differs).

---

## rule_violation

**File:** `rule_violation.yaml`
**Turns:** 15 | **Seed:** 101 | **adversarial_rate:** 0.7

Mixes 3 scripted grounding actions (`look around`, `take inventory`, `observe the room`,
`wait`) with 11 adversarial injects drawn from the `AdversarialBank` rule_violation
category. These probe conservation law enforcement: attempts to create gold from nothing,
duplicate the agent, grant invincibility, and similar physics-breaking commands.

**Primary D-12 metric:** `refusal_rate` (should be high — conservation violations must be
refused); `mutation_count` (should be 0 or low on refused turns).
**Expected profile:** Most adversarial turns produce refusals; scripted turns may succeed
depending on mechanic availability.

---

## repetition_loop

**File:** `repetition_loop.yaml`
**Turns:** 10 | **Seed:** 102 | **adversarial_rate:** 0.0

One scripted action (`walk east`) followed by 9 `repeat_last` injects, causing the same
action to be submitted 10 times in a row. Tests the `action_novelty` scoring metric
and verifies the engine handles repeated identical actions gracefully.

**Primary D-12 metric:** `action_novelty` (should trend toward 0 as repetitions accumulate).
**Expected profile:** First turn may succeed; subsequent repeats may be refused or
yield diminishing observation variety depending on engine/mechanic design.

---

## edge_case_stress

**File:** `edge_case_stress.yaml`
**Turns:** 20 | **Seed:** 103 | **adversarial_rate:** 0.0

20 `edge_case` injects cycling through: empty string `""`, a 500-character repeated
`"a"` string, and a special-character string with `?!@#$%^&*()<script>` content.
Tests that the engine does not crash on boundary inputs and handles empty actions
with a clean `no_viable_action` verdict.

**Primary D-12 metric:** Crash-free completion (the primary check is that the runner
finishes all 20 turns without exception); `mechanic_match_rate` (expected near 0).
**Expected profile:** All turns return a clean refusal or `no_viable_action`; no
unhandled exceptions at any point in the pipeline.

---

## mixed_chaos

**File:** `mixed_chaos.yaml`
**Turns:** 30 | **Seed:** 104 | **adversarial_rate:** 0.3

30 turns interleaving all four inject types (`adversarial`, `nonsense`, `edge_case`,
`repeat_last`) with agent-decided turns (`action: null`) and one scripted action at
start and end. Designed to simulate realistic adversarial pressure within an otherwise
normal playtest session.

**Primary D-12 metric:** `composite` score — the combined average across all five
metrics gives a holistic health signal for the engine under mixed adversarial load.
**Expected profile:** Agent-decided turns may produce coherent observations; inject
turns should mostly be refused cleanly; no cross-turn contamination (engine state
must remain consistent).

## Interpreting Playtest Reports

After running an adversarial scenario, inspect the JSON report in
`<universe>/playtest-reports/<run_id>.json`. Key fields:

- **`aggregate_scores.mechanic_match_rate`**: fraction of turns where a mechanic was matched
  (not yielded or refused). Lower is expected for adversarial scenarios.
- **`aggregate_scores.refusal_rate`**: inverse of refusals per N turns. Higher is better
  for adversarial scenarios where the engine is correctly refusing invalid actions.
- **`aggregate_scores.action_novelty`**: edit-distance variety across recent actions.
  `repetition_loop` should drive this toward 0.
- **`aggregate_scores.composite`**: average of all five metrics. A healthy engine under
  adversarial load typically scores 0.3–0.6 composite (lower than normal runs due to
  intentional refusal pressure).

Compare composite scores across runs to detect regressions. A sudden drop in
`observation_groundedness` on the `mixed_chaos` scenario may indicate a grounding
regression in the observer pipeline.
